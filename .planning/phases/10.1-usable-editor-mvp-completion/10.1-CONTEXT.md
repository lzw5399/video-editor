# Phase 10.1: Usable Editor MVP Completion - Context

**Gathered:** 2026-06-18
**Status:** Ready for planning
**Source:** User-directed urgent insertion before Phase 11

<domain>
## Phase Boundary

This phase closes the gap between a demo-like desktop workspace and a genuinely usable Jianying/CapCut-style MVP editor. It does not add retiming, effects, or transitions. It makes the already-built draft/material/timeline/render/text/keyframe foundations usable through the Electron desktop app.

The user-visible target is:

- import video/audio/image through a system file chooser, not typed paths;
- add imported materials to compatible timeline tracks;
- seek/scrub the playhead and see the real preview frame pixels;
- edit video/image/text transforms, text styles, audio volume, track mute, and SRT subtitles through Rust commands;
- keep all draft semantics Rust-owned and all derived media artifacts outside `.veproj/project.json`.

</domain>

<decisions>
## Locked Decisions

### Architecture

- Renderer UI must only emit generated command envelopes for draft/timeline/render semantics.
- Renderer must not directly mutate `draft.tracks`, `track.segments`, segment timeranges, `Segment.text`, `Segment.visual`, segment volume, track mute, undo/redo, keyframes, render graph, FFmpeg commands, export scripts, preview cache, waveform cache, or project canonical files.
- Electron main/preload may expose a narrow platform API for system dialogs. Renderer cannot import Electron, Node, `fs`, `path`, `child_process`, FFmpeg, or ffprobe.
- `window.videoEditorCore` remains the Rust command bridge. New platform-only APIs should be exposed separately so the command bridge stays stable.
- Persisted time semantics remain integer microseconds.

### File Import

- `导入素材` opens the OS file chooser.
- Supported extensions: video `mp4`, `mov`, `m4v`, `webm`; audio `mp3`, `wav`, `m4a`, `aac`; image `png`, `jpg`, `jpeg`, `webp`.
- After file selection, renderer calls `importMaterial` for each selected path and accepts material state only from Rust command responses.
- Typed path inputs may remain temporarily as advanced/test fallback, but the primary UX must not require manual path entry.

### Preview

- `requestPreviewFrame` returns a PNG artifact and the preview monitor must show the image itself inside the draft canvas ratio.
- The artifact path may still be shown in diagnostics, but not as the primary preview.
- When no visible frame exists, show a black/empty canvas state.
- Edits that change visible output clear stale preview state and request/re-enable the current playhead frame.

### Timeline And Selection

- Timeline must expose video/audio/text tracks, segment selection, move, trim, split, delete, undo, redo, and add-to-compatible-track.
- Playback head is a vertical Jianying-like line.
- Playhead can be changed by clicking the ruler, dragging the playhead line, entering a time, and previous/next frame controls.
- Playhead changes request preview for the current integer-microsecond time.
- Selection is visible in timeline, preview overlay, and inspector. No selection shows draft parameters.

### Video, Audio, Text, Subtitle

- Video/image/text visual changes use `updateSegmentVisual`.
- Audio volume and track mute use `setSegmentVolume` and `setTrackMute`.
- Text add/edit/style uses `addTextSegment` and `editTextSegment`.
- SRT import uses Rust-owned `importSubtitleSrt`; renderer must not parse SRT into cue segments.
- P0 waveform is a placeholder. Real waveform cache is P1 and must remain a derived artifact.
- P1 canvas drag/scale/rotate handles are explicitly deferred; this phase may prepare the overlay but should not fake direct semantic ownership in the renderer.

</decisions>
