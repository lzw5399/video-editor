<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Electron renderer -> preload/main | Renderer input is untrusted and must cross allowed IPC routes only. |
| Node-API transport -> shared runtime | JSON/Node values are transport data and cannot own semantics or resource metadata. |
| Desktop export route -> render/export runtime | Desktop adapters must not decide render/export behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-09 | Spoofing | Electron IPC | mitigate | Preserve `assertAllowedIpcSender` and explicit IPC method allow-list. |
| T-18-10 | Tampering | `bindings_node` project sessions | mitigate | Delegate registry/revision/interaction state to `editor_runtime`; source guard rejects binding-owned registry policy. |
| T-18-11 | Tampering | desktop export adapter | mitigate | Delegate export lifecycle to `editor_runtime::ExportService`; guard binding-owned FFmpeg/render graph/export registry behavior. |
| T-18-12 | Repudiation | product success evidence | mitigate | Keep no-product-fallback and Phase 18 guards in verification; artifact/CPU/mock/DOM evidence cannot satisfy success. |
| T-18-SC | Tampering | npm tooling | mitigate | Do not change `@napi-rs/cli`; any required change must stop for the checkpoint required by RESEARCH.md. |
</threat_model>
