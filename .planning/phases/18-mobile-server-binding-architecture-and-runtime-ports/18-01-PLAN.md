<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Adapter token -> Rust runtime | JavaScript/C/server callers can present opaque tokens but cannot be trusted to own or fabricate metadata. |
| Runtime session close -> resource registry | Unreleased resources must cascade deterministically and produce diagnostics. |
| Texture/frame import -> device context | Native resource handles are only valid for the recorded backend/device/session/generation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Elevation of Privilege | `HandleRegistry::resolve` | mitigate | Validate kind, owner session, generation, ref/lease state, and device identity before returning resource access. |
| T-18-02 | Tampering | `HandleRegistry::release` | mitigate | Return typed double-release/unknown-handle errors and never treat repeated release as success. |
| T-18-03 | Information Disclosure | frame/texture handles | mitigate | Store resource metadata in Rust and expose only opaque tokens to adapters. |
| T-18-04 | Denial of Service | session close | mitigate | Cascade release owned handles and bound outstanding leases; leak diagnostics identify unreleased resources. |
| T-18-SC | Tampering | package installs | mitigate | This plan adds no package-manager install. Later cbindgen usage relies on RESEARCH.md Package Legitimacy Audit and pinned 0.29.4. |
</threat_model>
