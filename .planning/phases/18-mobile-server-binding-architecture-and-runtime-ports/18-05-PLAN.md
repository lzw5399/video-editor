<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server CLI args -> project/export runtime | Paths and JSON input are untrusted and must be validated. |
| `.veproj` material URI -> filesystem | Bundle-relative material resolution must prevent traversal and machine-local leakage. |
| Server runtime -> FFmpeg executor | Runtime executes jobs and reports progress; it does not decide editing semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-17 | Tampering | server bundle path/material URI | mitigate | Use `project_store` open and URI classification; reject traversal and invalid material resolution. |
| T-18-18 | Information Disclosure | server material resolution | mitigate | Keep bundle-relative resolution explicit and avoid local absolute-path fallback as success. |
| T-18-19 | Repudiation | server export evidence | mitigate | Tests assert output file, metadata, progress, cancellation, and no Electron dependency. |
| T-18-20 | Denial of Service | server export jobs | mitigate | Use `task_runtime` scheduler/cancellation and bounded process execution patterns. |
| T-18-SC | Tampering | package installs | mitigate | No new external package install in this plan; FFmpeg runtime reuse must stay bundled/configured, not PATH fallback. |
</threat_model>
