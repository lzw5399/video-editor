<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| C caller -> Rust ABI | Raw pointers, buffer lengths, strings, and handles are untrusted. |
| Generated header -> external build | Header layout must match Rust exports. |
| Mobile-held token -> Rust registry | Future JNI/Swift callers may hold tokens but cannot own lifetime metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-13 | Tampering | C ABI buffers | mitigate | Validate pointers, lengths, UTF-8, output capacity, and provide explicit free APIs. |
| T-18-14 | Elevation of Privilege | `ve_handle_t` | mitigate | Resolve all handles through `editor_runtime::HandleRegistry` owner/generation/device checks. |
| T-18-15 | Denial of Service | FFI panic boundary | mitigate | Catch/avoid panics across FFI and return stable error codes plus bounded diagnostics. |
| T-18-16 | Repudiation | generated header | mitigate | Project-local pinned `cbindgen` drift script proves header matches exports. |
| T-18-SC | Tampering | `cbindgen` toolchain | mitigate | RESEARCH.md Package Legitimacy Audit approves `cbindgen`; script bootstraps version 0.29.4 locally and rejects mismatches. |
</threat_model>
