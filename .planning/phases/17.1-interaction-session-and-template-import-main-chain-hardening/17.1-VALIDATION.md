# Phase 17.1 Validation Matrix

**Scope:** High-frequency interaction sessions, template import main-chain integration, product preview evidence, and aggregate source/performance guards.

Phase 17.1 is valid only when each high-frequency product surface is either routed through Rust-owned interaction/navigation semantics or explicitly product-gated. DOM overlays, fallback preview frames, mock/artifact evidence, repeated canonical intent loops, and unsupported visible controls do not count as success.

## Source Guard Coverage

`scripts/phase17-1-source-guards.sh` rejects:

- High-frequency drag/slider/scrub handlers that call repeated canonical `executeProjectIntent` or `executeProjectTimelineIntent` loops.
- Reintroduced intent-name preview invalidation switches instead of `CommandDelta` consumers.
- Product E2E that accepts DOM-only ghost overlay movement or CSS transform changes as preview success.
- Unsupported visible high-frequency controls such as transition, fade, crop, or text-box handles without product gating/session backing.
- Missing inventory coverage for preview canvas, timeline, playhead/ruler/timecode, inspector, keyframe, and template report navigation classes.

## Audit Conclusion

Phase 17.1 is auditable from this file plus the aggregate commands. Supported high-frequency interactions use Rust-owned session updates with generation/sequence facts, no-save/no-undo provisional semantics, realtime preview evidence, stale rejection telemetry, and one canonical commit. Unsupported visible controls are intentionally product-gated and source-guarded until their Rust semantics, preview, export, and undo behavior exist.
