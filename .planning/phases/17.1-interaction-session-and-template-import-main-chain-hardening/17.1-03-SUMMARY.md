# Phase 17.1 Plan 03: Preview And Inspector Interaction Sessions Summary

**Preview canvas and inspector high-frequency edits now stream Rust-owned provisional state and commit once into canonical history.**

## Accomplishments

- Added renderer interaction-session plumbing from `App` through `WorkspaceShell` into `PreviewMonitor`, `Inspector`, and `FeaturePanel`.
- Preview move/rotate now begins a Rust `selectedSegmentVisual` interaction, streams RAF-coalesced updates, refreshes native preview with `interactionId`, and commits or cancels on pointer release/cancel.
- Inspector visual/text/audio high-frequency controls now use interaction sessions; normal numeric blur/discrete toggles remain canonical commands where appropriate.
- Rust project sessions now retain provisional draft/selection snapshots so realtime preview can render in-flight edits without mutating canonical revision.
- Product/workspace tests now assert provisional revision stability, one commit, no old 160ms debounce path, and native render-graph evidence for text/subtitle overlays.
