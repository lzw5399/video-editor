## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User panel input -> generated CommandEnvelope | Material/text/audio inputs become command payloads crossing to Rust. |
| Rust CommandResultEnvelope -> accepted renderer state | Successful command responses replace draft/state/selection; failed responses preserve prior state. |
| Renderer display selectors -> Draft | View-model selectors may read generated draft arrays but cannot mutate semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | Material/text/audio command builders | mitigate | Build envelopes from generated types only; do not hand-write parallel contracts or edit generated TS files. |
| T-04-07 | Tampering/Repudiation | Inspector rejected commands | mitigate | Preserve prior accepted state and show Chinese Rust error text from `CommandResultEnvelope.error`. |
| T-04-08 | Elevation of Privilege | Renderer material import path handling | mitigate | Renderer submits user-provided paths only through generated `importMaterial`; no renderer Electron/Node/filesystem imports. |
| T-04-09 | Tampering | Direct draft mutation in panels/inspector | mitigate | Use read-only selectors and command result replacement; Phase 04-04 source guards fail array mutation patterns. |
| T-04-10 | Information Disclosure | Missing/probe-failed material diagnostics | mitigate | Display bounded Rust diagnostic messages only; do not expose raw ffprobe JSON or filesystem APIs in renderer. |
