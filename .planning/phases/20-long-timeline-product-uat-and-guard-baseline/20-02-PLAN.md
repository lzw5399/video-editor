## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Playwright helper -> Rust CLI | Node test code invokes the Rust materializer with local paths. |
| packaged app -> bundled runtime discovery | Export proof depends on product runtime-reported ffprobe/ffmpeg paths. |
| product telemetry -> evidence JSON | Native observations and scheduler state are persisted as test evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T20-04 | Spoofing | preview evidence helper | mitigate | Require `renderGraphGpuComposited`, `renderGraphGpu`, diagnostic source `none`, visible pixel change, and no artifact preview-frame request increase. |
| T20-05 | Tampering | export evidence helper | mitigate | Use bundled runtime discovery and sampled semantic frames; do not treat file existence as sufficient success. |
| T20-06 | Information Disclosure | evidence bundles | mitigate | Store product-readable summaries separately from developer details and keep paths inside `test-results/phase20`. |
| T20-07 | Tampering | canonical summaries | mitigate | Normalize only canonical draft facts and explicitly exclude derived/runtime/cache/export facts. |
| T20-SC | Tampering | npm/cargo installs | mitigate | No new package installs; use Node built-ins, existing Playwright, and existing local Rust crates. |
</threat_model>
