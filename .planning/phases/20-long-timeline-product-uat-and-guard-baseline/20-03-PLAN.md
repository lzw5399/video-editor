## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generated `.veproj` -> packaged app | The packaged product opens a local generated project. |
| packaged app -> preview/export runtime | Product evidence crosses from Rust/native runtime into Playwright assertions. |
| background work -> UI interactions | Scheduler/export/probe/artifact/cache pressure competes with user operations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T20-08 | Spoofing | packaged preview proof | mitigate | Require production compositor evidence and visible pixel motion; reject artifact, DOM, native single-video, CPU, and first-frame-only proof. |
| T20-09 | Tampering | export proof | mitigate | Validate with bundled ffprobe/ffmpeg metadata and sampled semantic frames, not file existence alone. |
| T20-10 | Denial of Service | scheduler pressure | mitigate | Assert p95 queue latency, no rejected normal product work, no fallback success, and visible feedback budgets. |
| T20-11 | Repudiation | UAT failure evidence | mitigate | Write evidence bundles containing product-readable workflow/stage summaries plus developer telemetry and native observations. |
| T20-SC | Tampering | npm/cargo installs | mitigate | No new dependencies; use existing Electron, Playwright, Rust, and app runtime helpers. |
</threat_model>
