## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| project JSON -> Rust draft model | Untrusted `.veproj/project.json` data enters serde/migration/validation. |
| material registry -> image background reference | Canvas background references must only point at valid image materials. |
| generated contracts -> renderer | Renderer receives Rust-owned canvas types and command payload shapes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | `migrate_draft_json` / `validate_draft` | mitigate | Require `canvasConfig`; reject unknown fields, zero dimensions/fps, mismatched aspect presets, invalid colors, and invalid image material references. |
| T-07-02 | Spoofing | image background material IDs | mitigate | Validate that image background `materialId` exists and has image material kind before accepting the draft. |
| T-07-03 | Repudiation | schema/contracts | mitigate | Regenerate JSON Schema and TypeScript contracts from Rust and include fixture tests for positive and negative cases. |
| T-07-04 | Information Disclosure | coordinate docs | accept | Coordinate docs contain no secrets or paths; they only define public semantic math. |
| T-07-SC | Tampering | package installs | mitigate | No npm/Rust packages are installed in this plan per D-11; generated-contract drift remains the supply-chain check. |
</threat_model>
