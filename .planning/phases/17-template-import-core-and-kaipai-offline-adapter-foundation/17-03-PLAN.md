<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter output -> import plan | Adapter-controlled data becomes candidate canonical draft state. |
| import plan -> project session | Validated plan will later mutate `.veproj/project.json` through the session owner. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-09 | Tampering | `DraftImportPlan` validation | mitigate | Validate canonical draft shape and reject invalid material refs/timeranges before session mutation. |
| T-17-10 | Information Disclosure | Plan fields | mitigate | Keep raw formula/provenance evidence out of canonical plan fields that persist to project JSON. |
| T-17-11 | Tampering | Remote runtime refs | mitigate | Reject URL-style material refs in import-plan validation unless explicitly classified as diagnostic-only report evidence. |
| T-17-12 | Repudiation | Unsupported feature status | mitigate | Require returned report items for unsupported or approximated plan inputs. |
</threat_model>
