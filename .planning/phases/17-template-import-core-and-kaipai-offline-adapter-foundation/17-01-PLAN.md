## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| offline template bundle -> Rust adapter/import contracts | Untrusted JSON and paths eventually influence project resources and draft semantics. |
| report contract -> desktop UI | Product-facing report copy must not expose raw provider secrets or classify unsupported features as success. |
| source guard -> normal product build | Guard failures block provider-specific render semantics and fallback evidence from entering product paths. |

| T-17-02 | Information Disclosure | Report provenance fields | mitigate | Preserve external references only as bounded evidence; tests reject credential-like fixture/report keys in later plans. |
| T-17-03 | Tampering | Core/render/session source boundaries | mitigate | `scripts/phase17-source-guards.sh` blocks provider/raw formula/live API/Android dependencies outside adapter code. |
| T-17-04 | Repudiation | Product success evidence | mitigate | Guard links Phase 17 with no-product-fallback policy so mock/artifact/CPU/Android evidence cannot satisfy preview/export success. |
| T-17-SC | Tampering | package installs | mitigate | No package install occurs in this plan; downstream package additions must rely on the RESEARCH.md package legitimacy audit. |
