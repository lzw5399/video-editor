## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source bundle path -> `.veproj/resources` | Untrusted file references are copied into the project bundle. |
| resource metadata -> report/draft plan | Resource evidence can influence user-visible diagnostics and material refs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-05 | Tampering | Resource path canonicalization | mitigate | Reject traversal, absolute destination, symlink escape, and duplicate destination writes. |
| T-17-06 | Information Disclosure | Remote URLs and credentials | mitigate | Reject remote runtime refs and scan fixtures/resources for credential-like fields. |
| T-17-07 | Tampering | Resource integrity | mitigate | Validate declared SHA-256 with `sha2::Sha256` and report mismatches. |
| T-17-08 | Denial of Service | Resource copy set | mitigate | Bound each test fixture set and reject invalid entries before draft/session mutation. |
| T-17-SC | Tampering | `sha2` dependency | mitigate | Use the RESEARCH.md Package Legitimacy Audit verdict OK; commit Cargo.lock updates from Cargo. |
</threat_model>
