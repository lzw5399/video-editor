# Phase 15 - UI Design Contract

> Visual and interaction contract for production audio preview, DSP-facing audio controls, waveform display, and playback status. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none; existing hand-authored React + CSS renderer |
| Preset | not applicable |
| Component library | none |
| Icon library | Existing compact text/CSS symbols only; do not add a new icon dependency for this phase |
| Font | Inter, "PingFang SC", "Microsoft YaHei", ui-sans-serif, system-ui, sans-serif |

Sources: `AGENTS.md`, `.planning/REQUIREMENTS.md`, Phase 15 `CONTEXT.md`, Phase 15 `RESEARCH.md`, Phase 14 `UI-SPEC.md`, Jianying Pro reference notes, and current renderer files under `apps/desktop-electron/src/renderer/`.

shadcn gate result: `components.json`, Tailwind config, and PostCSS config are absent. Phase 14 already established the approved renderer design system as manual CSS in a compact Jianying-style desktop shell. Do not initialize shadcn or introduce registry components in Phase 15.

Phase 15 must preserve the five-zone editor workspace: top feature bar, left material/audio panel, center preview monitor, right inspector, and bottom timeline. It must add production-facing audio playback, buffering, waveform, and device status only. It must not add a debug audio graph dashboard, raw DSP telemetry panel, digital-human surface, landing page, broad visual redesign, or renderer-owned audio internals.

Visual hierarchy: preview transport and timeline playback state are the primary focal points for audio preview. Timeline waveform display is secondary and remains attached to audio segments. The Audio feature panel and inspector provide compact editing controls for segment/track audio properties. Device/status feedback is tertiary and appears as a small safe product label, not a diagnostics table.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Waveform bar gaps, status chip gaps, icon-button groups |
| sm | 8px | Audio control row gaps, waveform status padding, compact panel rows |
| md | 16px | Preview monitor padding, inspector section spacing, empty-state padding |
| lg | 24px | Audio panel grouped controls and compact device/status sections |
| xl | 32px | Primary/secondary action height, input height, playback status row ceiling |
| 2xl | 48px | Top feature bar, timeline toolbar, material thumbnail square |
| 3xl | 64px | Audio material row minimum height and maximum compact audio status section height |

Exceptions:

- Region dividers remain 1px.
- Scrollbars remain 4px.
- Timeline toolbar remains 48px high; track header width remains 160px.
- Preview and timeline icon buttons remain 28x28px, with 26px allowed only under the existing `max-width: 1199px` compact rule.
- Audio waveform display inside a segment is fixed at 14px high and must never resize the segment block, track row, or timeline panel.
- Waveform progress bars, if shown, are 6px high and must not change row height while progress updates.
- Existing desktop layout dimensions remain stable: 48px top bar, 336px left panel, 288px inspector, and 252px timeline at 1120x720 and 1280x800.

## Phase Scope

Phase 15 UI exposes production audio playback state, waveform display readiness, compact output-device selection/status, and user-facing audio edit controls. It does not expose the audio graph, sample buffers, native output handles, DSP evaluator internals, artifact storage internals, or export filter syntax.

Renderer may:

- Display Rust/binding-returned audio playback state, buffering state, safe output-device summaries, waveform readiness, display-ready waveform peaks, and safe preview/export parity labels.
- Hold local UI state for selected tab, pending button state, form values before submission, dropdown open state, and dismissed safe notices.
- Submit generated command envelopes for audio play, pause, stop, seek, cancel, device selection, waveform refresh, segment volume, pan, fade, and track mute.
- Format Rust-returned accepted values for display as percentages, left/right pan labels, seconds, or Chinese status text.

Renderer must not:

- Build audio graphs, mix buffers, gain curves, pan laws, fade envelopes, effect graphs, sample indices, output-device handles, FFmpeg filters, render graphs, cache keys, artifact roots, waveform blob paths, SQLite queries, fingerprints, dirty ranges, or timeline state.
- Derive canonical audio behavior from waveform or peak display data.
- Inspect native device handles, platform backend names, buffer sizes, stream configs, or low-level parity diffs in production UI.
- Directly mutate draft materials, tracks, segments, timeranges, keyframes, audio semantics, undo/redo, snapping state, waveform status, or artifact job state.

## Interaction Contract

### Preview Playback Controls

The existing preview transport and timeline transport remain the only primary playback controls. Phase 15 must make audio/video playback status understandable without adding a competing debug control surface.

Required behavior:

- When audio or video preview is available, the play button must be enabled and labeled `播放预览`; while running it changes to `暂停预览`.
- `停止预览` remains enabled while playback is running, even when another command is pending.
- If video preview is ready but audio is buffering, playback may start visually and show `音频缓冲中` as a warning status.
- If audio preview is unavailable but visual preview is available, the play button stays usable and the status line shows `音频暂不可用`.
- If no preview path is available, disabled controls must expose a recoverable label or title such as `预览暂不可用，请先添加素材` or `未找到输出设备`.
- Do not show gray, unlabeled, or ambiguous disabled playback buttons after preview is available.

Preview status line:

- Add or reuse one compact status chip for audio next to the existing preview status/canvas readouts.
- Show at most two audio facts at once: playback/buffering state and device/waveform warning if needed.
- Status text must come from Rust/bindings or a direct mapping of Rust-safe enum values; TypeScript must not infer graph/device internals.

### Audio Feature Panel

The Audio top feature panel keeps Jianying-style feature reduction. It should feel like an editing panel, not an audio engineering panel.

Required layout:

- Header: `音频` plus primary `添加音频`.
- Audio material selector: compact select using available audio material display names.
- Segment controls: `音量`, `声像`, `淡入`, `淡出`.
- Track controls: `轨道静音` / `取消轨道静音`.
- Output controls: one compact `输出设备` row with `系统默认` as the default option and safe device display names returned by Rust/bindings.
- Status row: one line for `音频就绪`, `音频缓冲中`, `未找到输出设备`, or `音频预览失败`.

Control rules:

- Display volume as user-facing percent, with 100% as neutral. Persisted units remain Rust-owned.
- Display pan as `左` / `居中` / `右` or a centered slider. Do not expose pan law, channels, or floating-point coefficients.
- Display fades as `淡入` and `淡出` durations. UI may collect values, but accepted values and validation come from Rust command responses.
- `淡入淡出` may remain visible as a capability chip only until commands are wired; once wired, it becomes editable controls.
- Avoid labels such as `毫音量` in production copy. Developer diagnostics may show raw units only when diagnostics are explicitly enabled.

### Inspector Audio Tab

The right inspector `音频` tab is the authoritative editing surface for a selected segment's accepted audio values.

Required fields when an audio-capable segment is selected:

| Field | UI Control | Copy |
|-------|------------|------|
| Volume | slider + numeric percent input | `音量` |
| Pan | centered slider or segmented control | `声像` |
| Fade in | numeric duration input | `淡入` |
| Fade out | numeric duration input | `淡出` |
| Track mute | checkbox/toggle | `轨道静音` |
| Volume keyframe | existing diamond keyframe affordance | `音量关键帧` |

Rules:

- Controls keep local draft form state until the user clicks `应用音频` or an established immediate command pattern is already in use.
- Accepted values update only after Rust-shaped command responses.
- Keyframe indicators remain compact diamonds and must not overlap waveform bars or segment titles.
- Unsupported future effects show a quiet `暂未接入` capability row; do not show effect graph internals.

### Timeline Waveform Display

Timeline audio segments must replace the deterministic placeholder stripe with real display waveform peaks when Rust/bindings provide bounded display-ready data.

Required states:

| State | Timeline Display |
|-------|------------------|
| No waveform payload | Existing stable placeholder stripe with `aria-label="音频波形占位"` |
| Waveform pending | Placeholder stripe plus warning title/copy `波形生成中` where space allows |
| Waveform ready | Fixed-height waveform bars with `aria-label="音频波形"` |
| Waveform failed | Placeholder stripe plus destructive title/copy `波形生成失败`; retry action stays in panel/resource task UI, not inside the segment |

Display rules:

- Waveform bars stay within the existing segment block and remain 14px high.
- Bar count may adapt to segment width, but bars must use stable min/max dimensions and never change track height.
- Ready waveform bars use muted/success green tones, not accent cyan across the whole segment.
- Accent cyan is reserved for selected segment outline, playhead, and focused/active states.
- Waveform data is display-only and must not drive audio preview, export mixdown, gain, pan, fades, or keyframes.
- Clicking an audio segment selects it; waveform bars do not add separate click targets in Phase 15.

### Audio Preview And Export Parity Status

Parity feedback is user-safe and compact.

- If Rust reports preview/export audio parity as normal, default UI does not need to show a separate parity label.
- If Rust reports a classified warning, show `音频导出可能与预览不同` in the export panel or preview status line with warning color.
- Do not show raw parity diffs, sample summaries, FFmpeg filter text, backend names, or native stream details in production UI.

### Device And Buffer Status

Device and buffering state are recoverable product states.

- Default device option label: `系统默认`.
- Device-unavailable state label: `未找到输出设备`.
- Recoverable action: `选择输出设备` or `重试音频`.
- Buffering state label: `音频缓冲中`.
- Cancellation state label: `音频请求已取消`.
- Stale/canceled generation details must be compressed to safe user copy such as `声音已同步到最新播放头`; do not expose generation IDs.
