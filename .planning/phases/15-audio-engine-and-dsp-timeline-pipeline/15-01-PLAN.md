<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer command payload -> Rust draft_commands | Untrusted UI values for gain, pan, fade, and effect-slot classification enter canonical draft mutation. |
| Draft JSON -> audio DSP consumers | Persisted `.veproj/project.json` is loaded into Rust model semantics and later consumed by audio_engine/export. |
| Generated contracts -> TypeScript | Schema and generated TS carry safe command transport fields into renderer code. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01-01 | Tampering | `update_segment_audio` | mitigate | Validate gain, pan, fade, track lock, segment identity, and timeranges in Rust before mutation; invalid edits return atomic errors. |
| T-15-01-02 | Tampering | Audio dirty deltas | mitigate | Emit Rust-owned audio/export/waveform/graph/cache dirty domains from accepted command responses so renderer cannot invent invalidation facts. |
| T-15-01-03 | Information Disclosure | Generated contracts | mitigate | Schema tests reject waveform paths, SQLite names, cache keys, fingerprints, native handles, raw buffers, and FFmpeg filter strings in binding-visible audio contracts. |
| T-15-01-04 | Denial of Service | Fade/effect-slot values | mitigate | Bound integer fade durations, effect-slot counts, and gain/pan units; reject overlarge payloads before draft mutation. |
</threat_model>
