<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binding export API -> scheduler | User starts/cancels export through typed Node-API, but scheduler owns admission and resource policy. |
| Scheduler -> `media_runtime` | Scheduler invokes FFmpeg execution through the runtime executor and cancellation token. |
| Export completion -> product status | Export status transitions must respect cancellation and validation telemetry. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03A | Denial of Service | export admission | mitigate | Export resource budget, bounded queue, classified rejection, and interactive-lane starvation tests. |
| T-16-03B | Tampering | export cancellation/status | mitigate | Scheduler cancellation token is the source of truth; terminal statuses cannot be overwritten by late FFmpeg events. |
| T-16-03C | Tampering | FFmpeg execution boundary | mitigate | Keep argument-array process execution in `media_runtime`; UI and Electron never construct FFmpeg commands. |
| T-16-03D | Repudiation | export progress/status | mitigate | Scheduler telemetry records queued, running, cancelled, rejected, failed, and validation phases with job IDs. |
</threat_model>
