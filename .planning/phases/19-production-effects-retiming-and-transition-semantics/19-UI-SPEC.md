## Phase 19 Surface Contracts

### Resource Panel

Effects, transitions, filters, and adjustments must use the existing `showcase-panel-layout` structure: left rail, right card grid, compact cards, no nested cards.

Enabled card contract:

| Element | Contract |
|---------|----------|
| Card size | Stable grid cell; content must not resize panel on hover/selection |
| Preview | Abstract but product-specific swatch/thumbnail; no stock imagery |
| Title | Concrete effect/transition/filter name, max one line |
| Status chip | `预览支持` / `导出支持` / `预览降级` / `暂不支持` |
| Primary action | `应用效果`, `应用滤镜`, or `添加转场` |
| Drag support | Supported effect/filter cards may drag to selected segment or compatible track; transition cards may drag to a valid segment boundary |

Disabled card contract:

- Keep disabled cards visible for known-but-unsupported capability categories.
- Disable pointer action, use `aria-disabled="true"`, and show `暂不支持` or `导出暂不支持`.
- Do not use disabled cards as successful product evidence.

### Inspector

Add Phase 19 controls in compact sections. Controls must fit within the existing 288px inspector column.

| Section | Controls | Contract |
|---------|----------|----------|
| `变速` | Segmented preset buttons `0.5x`, `1x`, `2x`, `自定义`; slider; numeric multiplier; `音频跟随变速` toggle | Display decimal multipliers, but command payloads must be integer/rational; no UI-owned time mapping |
| `效果` / `调节` | Applied effect list, enable toggle, remove button, parameter sliders + numeric inputs | Sliders use provisional Rust interaction sessions; numeric blur/Enter commits |
| `滤镜` | Applied filter selection and strength slider | Strength display as percent; command payload uses typed integer units |
| `蒙版` | Shape selector, feather/opacity sliders, invert toggle, reset action | Preview overlay handles are ghost/proxy only until Rust accepts provisional state |
| `混合` | Blend mode select, opacity slider, support chip | Unsupported modes disabled with capability reason |
| Capability | Compact preview/export chips | Always show whether preview and export are supported, degraded, or unsupported for the selected effect/transition |

Use familiar control types:

- Segmented controls for speed presets and mask shapes.
- Toggles for enable/disable, audio-follow-speed, invert mask.
- Sliders plus numeric inputs for multiplier, strength, blur, opacity, feather, duration.
- Color swatches only for color-adjustment values.
- Menus/selects for blend modes and capability-backed preset sets.
- Icon-only buttons with `title` and `aria-label` for remove, reset, keyframe, add, and delete.

### Timeline

Timeline row heights and the `128px` header width must not change.

| Affordance | Visual Contract | Interaction Contract |
|------------|-----------------|----------------------|
| Transition | Boundary pill/brace centered between adjacent visual segments; cyan when selected, amber when degraded; label shows `叠化 · 0.5s` style duration | Drag left/right duration handles through a Rust interaction session; invalid boundaries show inline state, not a fake transition |
| Constant speed | Compact speed badge inside selected segment, e.g. `1.5x`; badge uses cyan only when selected | Inspector is primary edit path; timeline speed handle may appear only after Rust supports retime interaction payloads |
| Speed curve | Small retime lane/diamonds inside selected segment only | Disabled unless curve semantics exist; do not expose editable curve points as local-only UI |
| Mask/effect markers | Small top-edge marker or chip on segments with applied effects/masks | Click selects segment and inspector section; no direct mutation from marker click |
| Retime/transition ghost | Semi-transparent overlay during drag, removed on commit/cancel | Ghost must be marked as provisional state and must not persist in draft state |

Existing trim handles, keyframe markers, and playhead controls keep their current visual priority. Transition handles must not obscure trim handles; if both exist at an edge, trim remains closest to the segment edge and transition sits above or below as a separate layer.

### Preview

Preview controls remain operational and compact. New Phase 19 overlays must follow these rules:

- Selection/mask/transition overlays are UI-local ghost/proxy feedback only.
- The native preview surface remains the product evidence for effects/transitions/retiming.
- Unsupported states use compact product copy such as `当前效果暂不支持预览`; do not expose backend diagnostics unless developer diagnostics are enabled.
- Mask handles use visible 10-16px handles with a 24px effective hit target.
- Blend/effect preview differences must be visible through GPU preview before the control can be treated as enabled.

## Interaction Contracts

Rust-owned interaction sessions are mandatory for all high-frequency Phase 19 manipulation.

| Interaction | Required Route |
|-------------|----------------|
| Effect strength slider drag | `beginProjectInteraction` → coalesced `updateProjectInteraction` → `commitProjectInteraction` / `cancelProjectInteraction` |
| Filter strength slider drag | Same session route |
| Mask handle drag | Same session route |
| Blend opacity slider drag | Same session route |
| Transition duration drag | Same session route |
| Timeline retime/speed handle drag | Same session route |
| Keyframe point drag | Existing `keyframeEdit` interaction pattern |

Rules:

- Coalesce pointer/slider updates with `requestAnimationFrame`; do not send one save/revision/undo per mousemove.
- UI may display provisional ghost state immediately, but committed product state must come from Rust response/generation.
- Commit on pointer up, Enter, or blur where appropriate. Cancel on Escape, pointer cancel, or invalid session state.
- Pending command state disables new edits but does not resize controls.
- Keyboard support must exist for all non-pointer-only controls: arrow keys for sliders, Enter to commit text/numeric fields, Escape to cancel in-progress interactions where focus remains in the control.
- Use integer microseconds, frame indices, rational frame rates, integer millis, or typed enum values in command payloads. Display decimals are formatting only.

Forbidden in renderer code:

- FFmpeg filter strings or export command construction.
- Source-to-target time mapping, transition overlap validation, effect evaluation, fallback selection, dirty-range/cache semantics.
- Persisting private Jianying/Kaipai IDs as internal effect semantics.
- DOM/CSS/canvas overlay effects counted as product preview success.

## Accessibility And Input

- Every icon-only button must have a Chinese `aria-label` and `title`.
- Cards that behave as buttons must be `button` elements or expose equivalent keyboard activation.
- Disabled unsupported cards must use `disabled` or `aria-disabled="true"` and must not be focus traps.
- Inspector tabs keep `role="tablist"` / `role="tab"` and `aria-selected`.
- Timeline playhead and retime handles expose slider semantics where keyboard manipulation is supported.
- Transition boundary handles need labels such as `调整叠化转场时长`.
- Live status for interaction commit/error uses `aria-live="polite"` in the existing compact status area, not a new floating toast system.

## Out Of Scope UI Constraints

- Do not design mobile app UI in Phase 19.
- Do not design server-rendering UI beyond capability/report copy visible in the desktop editor.
- Do not expose proprietary Jianying/Kaipai effect marketplace, native effect IDs, or full proprietary effect parity as product controls.
- Do not introduce a landing page, marketing hero, debug dashboard, or generic admin panel.
- `AI效果` must not become the destination for first-party production effects unless the label is explicitly changed to non-AI product terminology.

## Implementation Review Gates

The later implementation and UI audit should verify:

- 1120x720 and 1280x800 screenshots show no overlap, clipping, or text overflow in top bar, left resource panel, inspector, preview, and timeline.
- Effects/transitions/filters remain unavailable unless backed by Rust capability registry, preview/export support or explicit degraded diagnostics, and product E2E evidence.
- All Phase 19 high-frequency interactions use project interaction sessions and do not create undo/save/revision entries on every pointer move.
- Renderer source guards block FFmpeg strings, renderer-owned retime math, renderer-owned transition overlap decisions, and unsupported controls that appear enabled.
- Product preview evidence comes from the native render-graph GPU preview path, not DOM/CSS overlays or artifacts.
