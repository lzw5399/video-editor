<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fixture JSON -> Rust/schema validation | Test fixtures simulate user-controlled command payloads. |
| testkit -> FFmpeg/ffprobe process | Test harness launches local binaries with generated inputs. |
| process metadata -> assertions | ffprobe JSON output is parsed into Rust assertions. |
| CI runner -> repo gates | CI installs test tools and executes unified commands. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of privilege | Electron smoke tests | mitigate | Keep smoke tests using the narrow preload API from 01-01; do not add raw IPC access for test convenience. |
| T-01-02 | Tampering | `fixtures/draft/invalid-unknown-field.json` and schema validation | mitigate | Validate unknown-field rejection through serde and JSON Schema drift gates. |
| T-01-03 | Spoofing/Tampering | FFmpeg/ffprobe in render smoke | mitigate | Use 01-02 discovery and version probes before smoke execution; CI installs known tools but app does not bundle them. |
| T-01-04 | Denial of service | FFmpeg/ffprobe smoke logs | mitigate | Reuse bounded stderr summaries from media_runtime; do not dump unbounded process output into assertions. |
| T-01-05 | Tampering/Elevation of privilege | Render smoke FFmpeg invocation | mitigate | Use process argument arrays through runtime/testkit helpers; do not create shell-concatenated FFmpeg command strings or UI-authored commands. |
| T-01-SC | Tampering | CI and package installs | mitigate | Use only packages approved in 01-RESEARCH.md Package Legitimacy Audit and pinned versions in lockfiles; CI runs lockfile-based installs. |
</threat_model>
