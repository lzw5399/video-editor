## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service crates -> OS/runtime | Runtime and filesystem abstractions eventually cross into local process and filesystem APIs. |
| semantic crates -> service crates | Pure draft/timeline semantics must remain isolated from platform-dependent service traits. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of privilege | service-boundary trait placement | mitigate | Keep platform traits only in consuming service crates per D-18 through D-20. |
| T-01-02 | Tampering | pure semantic crate dependencies | mitigate | Verify pure crates do not import `FfmpegExecutor`, `PlatformFileSystem`, or `PreviewRenderer`. |
| T-01-03 | Repudiation | FFmpeg distribution scope | mitigate | Document that Phase 1 does not distribute FFmpeg per D-12, avoiding implicit licensing obligations. |
| T-01-SC | Tampering | cargo dependency additions | mitigate | Do not add new runtime dependencies beyond packages approved in 01-RESEARCH.md. |
</threat_model>
