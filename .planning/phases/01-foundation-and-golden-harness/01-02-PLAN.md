<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env/PATH -> media_runtime | User-controlled paths select process binaries to probe. |
| media_runtime -> OS process | Rust launches FFmpeg/ffprobe candidates. |
| process stderr/stdout -> error envelope | External binary output crosses into application logs/UI-ready errors. |
| command envelope -> runtime probe | Renderer-visible command request triggers runtime discovery. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of privilege | Electron preload/main IPC | mitigate | Preserve 01-01 narrow IPC surface; runtime probe is callable only through typed `execute_command`, not arbitrary native method invocation. |
| T-01-02 | Tampering | `CommandEnvelope` runtime probe command | mitigate | Keep serde `deny_unknown_fields`, schema tests, and unsupported-command errors after adding the runtime probe command. |
| T-01-03 | Spoofing/Tampering | `VE_FFMPEG_PATH`, `VE_FFPROBE_PATH`, PATH lookup | mitigate | Probe explicit paths with `-version`, record checked path/source, and surface structured errors. |
| T-01-04 | Denial of service | FFmpeg/ffprobe stderr/stdout | mitigate | Bound stderr/stdout summaries with `MAX_STDERR_SUMMARY_BYTES` before returning errors. |
| T-01-05 | Tampering/Elevation of privilege | FFmpeg process execution | mitigate | Use `std::process::Command` with argument arrays; forbid shell concatenation and UI-authored FFmpeg commands. |
| T-01-SC | Tampering | cargo dependency additions | mitigate | Use only 01-RESEARCH.md approved packages: `which`, `thiserror`, and existing workspace dependencies; do not add unaudited packages. |
</threat_model>
