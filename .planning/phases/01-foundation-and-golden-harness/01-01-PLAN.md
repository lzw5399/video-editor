<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer -> preload -> Electron main | Untrusted renderer code crosses into privileged desktop APIs through the preload surface. |
| Electron main -> Node-API addon | JavaScript calls native Rust functions and receives typed envelopes. |
| command JSON -> Rust serde model | User-controlled command payloads are deserialized into Rust-owned command contracts. |
| package manager -> build scripts | npm/crates dependencies execute build/install behavior during setup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of privilege | `apps/desktop-electron/src/preload/index.ts` | mitigate | Expose only `ping`, `version`, and `executeCommand` through `contextBridge`; do not expose raw `ipcRenderer` or arbitrary channel invocation. |
| T-01-02 | Tampering | `draft_model::CommandEnvelope` | mitigate | Use serde `deny_unknown_fields`, generated JSON Schema, and negative tests for unknown fields. |
| T-01-03 | Spoofing/Tampering | FFmpeg path handling | transfer to 01-02 | No FFmpeg execution in this plan; 01-02 owns env/PATH discovery, version probing, checked paths, and structured errors. |
| T-01-04 | Denial of service | Native binding load and command errors | mitigate | Return bounded envelope errors from `nativeBinding.ts`; do not stream unbounded native errors into renderer state. |
| T-01-05 | Tampering/Elevation of privilege | Renderer/media boundary | mitigate | Renderer code must not construct FFmpeg commands; no FFmpeg command strings are introduced in this plan. |
| T-01-SC | Tampering | npm/cargo installs | mitigate | Use only packages approved in 01-RESEARCH.md Package Legitimacy Audit; do not add unaudited packages. |
</threat_model>
