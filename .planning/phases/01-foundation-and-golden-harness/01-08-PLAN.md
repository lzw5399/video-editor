<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer -> preload -> Electron main | Untrusted renderer code crosses into privileged desktop APIs through preload. |
| Electron main -> Node-API addon | JavaScript calls native Rust binding functions. |
| generated TypeScript -> runtime payload | UI-facing types represent Rust-owned command contracts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of privilege | `apps/desktop-electron/src/preload/index.ts` | mitigate | Expose only `ping`, `version`, and `executeCommand`; no raw `ipcRenderer` or arbitrary channel invocation. |
| T-01-02 | Tampering | renderer command payload | mitigate | Use generated TypeScript contracts from Plan 01-06 and Rust validation from `draft_model`. |
| T-01-03 | Information disclosure | native binding load errors | mitigate | Return standardized bounded envelope errors from `nativeBinding.ts`. |
| T-01-04 | Tampering/Elevation of privilege | renderer/media boundary | mitigate | Grep gate forbids FFmpeg/ffprobe references in Electron source; UI must not construct FFmpeg commands. |
| T-01-SC | Tampering | npm dependency additions | mitigate | Use only npm packages approved in 01-RESEARCH.md Package Legitimacy Audit. |
</threat_model>
