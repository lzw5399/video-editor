- **Fix:** Installed `ffmpeg-full 8.1.2`, removed old `ffmpeg`, wrote PATH/VE_* shell config, and made the desktop gate prefer `ffmpeg-full`.
- **Files modified:** `scripts/phase15-3-desktop-gate.sh`
- **Verification:** Focused real workflow passed 2/2; full aggregate passed 71/71 desktop E2E plus cargo check.

**2. [Rule 1 - Bug] Preview compiler capabilities defaulted to all available**
- **Found during:** same real workflow investigation
- **Issue:** Export used runtime-derived compiler capabilities, but preview service config used `PreviewServiceConfig::new`, which defaults to test-like all-available capabilities. Missing text filters were not classified at compile time.
- **Fix:** Preview service config now uses `compiler_capabilities_from_runtime(runtime)`.
- **Files modified:** `crates/bindings_node/src/lib.rs`, `crates/bindings_node/src/preview_export_service.rs`
- **Verification:** `cargo check -p bindings_node --locked`; focused real workflow; full aggregate gate.
