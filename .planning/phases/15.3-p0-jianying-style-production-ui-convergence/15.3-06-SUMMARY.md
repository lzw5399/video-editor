**Jianying-style timeline controls with compact toolbar actions, pointer segment move/trim, and product E2E coverage for normal editor interactions**

## Accomplishments

- Replaced the default numeric timeline command strip with compact accessible controls for undo, redo, play/pause, split selected/current, delete, add-track, snapping status, and zoom.
- Added pointer-based segment drag move and edge trim handles that submit existing `moveSegment` and `trimSegment` command envelopes using integer microsecond deltas.
- Updated product E2E coverage so timeline seek, playhead drag, split, add-track, snapping, delete, undo/redo, zoom, move, and trim use visible product interactions rather than debug spinbuttons.

## Decisions Made

- Default product UI removes the numeric `播放头`, `移动`, `分割`, and `裁剪` spinbuttons instead of leaving them visible as primary controls.
- Timeline pointer interactions are intent translators only: they compute integer microsecond deltas and submit existing command callbacks; Rust responses remain authoritative for snapping and accepted state.
- Add-track remains grouped as compact visible track-type buttons, keeping the toolbar accessible without adding an uncovered menu path.
