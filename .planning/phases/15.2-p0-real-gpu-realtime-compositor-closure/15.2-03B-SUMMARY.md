# Phase 15.2 Plan 03B: Decoded Native Texture Import Summary

**Generation-checked decoded texture leases now gate media IO handoff and feed registered WGPU texture resources into the render-graph compositor without fallback success.**

## Accomplishments

- Added RED tests requiring texture descriptors to preserve owner session, device identity, color, and generation facts, plus fail-closed behavior for metadata-only texture handles.
- Added `NativeTextureLeaseRegistry` in `media_runtime`, with validation for registered handle identity before leases can be resolved.
- Wired `MediaIoFrameProvider` so compatible decoded texture frames still fail closed unless a live native texture lease is registered.
- Connected macOS native decode sessions to the lease registry when CoreVideo/Metal texture interop produces retained texture resources.
- Taught the WGPU compositor and texture cache to resolve registered leases and sample registered `wgpu::Texture` resources directly, while rejecting unsupported native resource kinds without CPU or FFmpeg fallback.

## Decisions Made

- Media IO does not treat a compatible texture descriptor as product-ready until the descriptor resolves against a live native texture lease.
- The compositor accepts registered `wgpu::Texture` lease resources as the supported sampleable import path; platform-native resources that are not represented as WGPU textures remain unavailable diagnostics.
- `presentable_frame_count` now increments only after final frame-provider conversion succeeds, so missing leases cannot inflate successful product-input telemetry.
