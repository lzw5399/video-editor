# Phase 15.2 Plan 05: Product E2E Matrix Summary

Plan 05 is complete. The desktop editor now has normal-user Playwright coverage for the mainstream supported subset, plus a real save/reopen/export workflow that uses repo-owned video, image, audio, and text fixtures in both dev and packaged app runs.

## Accomplishments

- Added product E2E matrix coverage for normal user editing paths, including import, timeline add/edit, seek/scrub/playback, transforms, text, audio, delete, undo/redo, and export-facing state.
- Tightened product preview gates so `requestPreviewFrame`, artifact fallback, runtime-frame diagnostics, native bridge evidence, and mock evidence cannot satisfy product playback.
- Added `openProjectBundle` and `saveProjectBundle` command contracts and Node binding handlers over `project_store`.
- Autosaved accepted timeline/canvas draft changes through Rust command envelopes and reopened `.veproj/project.json` in dev and packaged E2E.
- Switched real workflow tests to committed repo media fixtures instead of generating or depending on external ad-hoc media.
- Extended real workflow to import video, image, and audio; add video, text, audio, and image segments; request preview artifacts; export; then reopen and verify persisted materials, segments, and text inspector state.
- Mapped export compiler capabilities from the actual desktop FFmpeg runtime so missing H.264/AAC/text/font support fails explicitly.
- Fixed FFmpeg compiler image-layer export timing: image materials now become timed still layers and visual overlays are enabled only in their target time windows, avoiding 1-frame or prematurely truncated exports.
