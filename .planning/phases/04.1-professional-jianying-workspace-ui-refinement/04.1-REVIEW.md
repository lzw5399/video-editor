## Warnings

### WR-01: Track-header mute control is rendered as a disabled shell despite existing mute command support

**Classification:** WARNING
**File:** `apps/desktop-electron/src/renderer/workspace/Timeline.tsx:326`
**Issue:** The timeline renders lock, visibility, and mute state controls through the same disabled `TrackStateButton`. Lines 326-328 create the mute button, and lines 383-389 hard-disable every track state button. The phase contract allows lock/visibility placeholders, but explicitly requires mute to preserve existing command behavior for audio/selected tracks. This leaves a visible mute affordance in the track header that cannot invoke the existing `setTrackMute` command path.
**Fix:** Add a dedicated enabled mute button path and pass a timeline callback from `WorkspaceShell`/`App`, while keeping lock and visibility disabled until their command semantics exist.

```tsx
<TrackStateButton label={`锁定状态：${row.lockLabel}`} symbol="锁" disabled />
<TrackStateButton label={`可见状态：${row.visibilityLabel}`} symbol={row.track.kind === "audio" ? "听" : "眼"} disabled />
<TrackStateButton
  label={`静音状态：${row.muteLabel}`}
  symbol="静"
  active={row.track.muted}
  onClick={() => onSetTrackMute?.(row.track.trackId, !row.track.muted)}
/>
```

### WR-02: Inspector tabs change selected styling but do not control the displayed panel content

**Classification:** WARNING
**File:** `apps/desktop-electron/src/renderer/workspace/Inspector.tsx:127`
**Issue:** The inspector exposes a `role="tablist"` and updates `activeTab` on click, but the rendered selected/no-selection content at lines 142-381 is independent of `activeTab`. Users and assistive technologies get real tabs whose selected state changes while all sections remain visible in the same scroll area. That is misleading tab behavior and the current tests only assert the tabs are visible.
**Fix:** Either render a matching `tabpanel` per active tab, or remove tab semantics and present these controls as a non-tab segmented filter until tab-specific content is implemented.

```tsx
{selected === null ? (
  <DraftParameterPanel workspace={workspace} />
) : activeTab === "画面" ? (
  <PicturePanel selected={selected} />
) : activeTab === "音频" ? (
  <AudioPanel selected={selected} volume={volume} onVolumeChange={setVolume} />
) : (
  <DeferredInspectorTab tab={activeTab} />
)}
```

### WR-03: Import-command Playwright coverage can pass if the import command path is broken

**Classification:** WARNING
**File:** `apps/desktop-electron/tests/workspace.spec.ts:400`
**Issue:** The test named `material import uses the same draft command guard as timeline edits` clicks `添加片段` and `导入素材` back-to-back, then asserts only `addSegment` was recorded at lines 407-410. Because the first click puts a command in flight, this proves the import attempt was suppressed, not that `导入素材` routes through `window.videoEditorCore.executeCommand`. A regression that disconnects the import button from `importMaterial` would still pass.
**Fix:** Split this into two tests: one fresh-state test that clicks `导入素材` and expects an `importMaterial` execute call, and a separate concurrency guard test that verifies a second command is blocked while one is pending.

```ts
await spyExecuteCommandCalls(app, page);
await page.getByRole("button", { name: "导入素材" }).click();
await expectCommandCall(app, "importMaterial");
```

### WR-04: Source guard misses bracket/index forms of direct renderer mutation

**Classification:** WARNING
**File:** `scripts/phase4-source-guards.sh:42`
**Issue:** The direct mutation guard catches `.tracks =`, `.tracks.push(...)`, `track.segments =`, and array mutator calls, but it does not catch direct index or bracket mutations such as `draft.tracks[0] = nextTrack`, `workspace.draft["tracks"].push(track)`, or `track.segments[0] = segment`. Those are still direct renderer-owned draft/timeline mutations and violate UI-12, but this guard would miss them.
**Fix:** Prefer an AST-based guard for assignment/update expressions against `tracks`, `segments`, timeranges, undo/redo, and snapping. If staying with regex, add index/bracket forms and keep command-helper payload construction excluded intentionally.

```bash
fail_if_matches \
  "renderer must not mutate Draft.tracks or Track.segments by index or bracket access" \
  '(?:draft|current|nextDraft|workspace\.draft)(?:\.tracks|\["tracks"\])\s*\[[^]]+\]\s*=|(?:track|candidate)(?:\.segments|\["segments"\])\s*\[[^]]+\]\s*=|(?:\.tracks|\["tracks"\])\.(?:push|pop|shift|unshift|splice|sort|reverse)\s*\(|(?:\.segments|\["segments"\])\.(?:push|pop|shift|unshift|splice|sort|reverse)\s*\(' \
  "${renderer_files[@]}"
```
