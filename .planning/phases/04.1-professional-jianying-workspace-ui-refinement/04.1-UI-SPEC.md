# Phase 04.1 — UI Design Contract

> Visual and interaction contract for Phase 04.1. Generated by gsd-ui-researcher for the professional Jianying-style workspace refinement.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none; existing hand-authored React + CSS renderer |
| Preset | not applicable |
| Component library | none |
| Icon library | Prefer `lucide-react` if a dependency is added; otherwise generic CSS/text symbols are allowed only when they remain compact and accessible |
| Font | Inter, "PingFang SC", "Microsoft YaHei", ui-sans-serif, system-ui, sans-serif |

Source: `AGENTS.md`, Phase 04.1 `CONTEXT.md`, existing `apps/desktop-electron/src/renderer/styles.css`, and `package.json`.

This phase must refine the existing `WorkspaceShell`, `FeaturePanel`, `PreviewMonitor`, `Inspector`, `Timeline`, `viewModel`, and `styles.css` surface. Do not introduce a landing page, dashboard metaphor, card-heavy marketing layout, or alternate editor shell.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, track-row inner gaps, status-dot gaps, ruler ticks |
| sm | 8px | Compact button gaps, input gaps, segment padding, small panel padding |
| md | 16px | Default side panel padding, section padding, major toolbar group gaps |
| lg | 24px | Preview monitor outer padding at 1280px and wider |
| xl | 32px | Reserved for larger empty-state vertical breathing room only |
| 2xl | 48px | Top feature bar height and fixed square touch/click rhythm |
| 3xl | 64px | Maximum top category item width at 1280px and wider |

Exceptions:

- Workspace dividers use 1px borders/gaps to separate editor regions.
- Timeline playhead uses 2px width and 12px handle because it is a precision marker.
- Status dots use 8px circles.
- Inputs and compact action buttons use 28px or 32px heights; icon-only toolbar buttons use 32px visual size with accessible labels.
- Preview canvas uses `aspect-ratio: 16 / 9` with fixed min/max bounds instead of spacing tokens.

## Typography

Use exactly these sizes and weights for visible workspace UI:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Metadata / captions | 12px | 400 | 1.4 |
| Body / controls | 13px | 400 | 1.5 |
| Panel heading / selected labels | 14px | 600 | 1.3 |
| Workspace title / monitor title | 18px | 600 | 1.2 |

Typography rules:

- Letter spacing is `0`.
- Do not scale font size with viewport width.
- Chinese UI copy must remain single-line in toolbar buttons where possible; truncate file/material names with ellipsis, not control labels.
- Use `font-weight: 600` only for panel headings, selected state labels, primary CTA text, and important segment names.
- Do not introduce hero/display typography. This is an operational editor.

## Phase Scope

Phase 04.1 upgrades the already-working Phase 04 shell into a denser professional desktop editor UI. It must preserve:

- React + TypeScript renderer.
- Command-only Rust integration through `window.videoEditorCore.executeCommand`.
- Simplified Chinese visible copy.
- Existing MVP material, text, audio, inspector, and timeline behaviors.
- Deferred but visible unsupported categories.

Phase 04.1 must not implement:

- Real preview frame rendering.
- Export execution.
- Render graph construction.
- FFmpeg or ffprobe command construction in renderer code.
- Preview cache, waveform generation, thumbnail generation, or export script semantics.
- Renderer-owned draft mutation, undo/redo stacks, snapping semantics, or timeline timerange edits.

## Workspace Layout Contract

The first screen is the editor workspace itself. The layout has five fixed regions:

| Region | Position | Desktop Contract |
|--------|----------|------------------|
| Top feature area | Full width, first row | 48px height; product mark at left; 11 icon-plus-text category entries visible at 1280x800 and 1120x720 |
| Left resource/function panel | Upper left | 300px at 1280px; 272px at 1120px; split into 56px category tree plus content area |
| Center preview shell | Upper center | `minmax(420px, 1fr)`; black 16:9 monitor canvas centered within titled shell |
| Right inspector | Upper right | 300px at 1280px; 272px at 1120px; primary tabs plus compact parameter rows |
| Bottom timeline | Full width, last row | 252px at 1280x800; 220px at 1120x720; compact toolbar, ruler, track list |

Grid requirements:

- At `1280x800`: use `grid-template-columns: 300px minmax(420px, 1fr) 300px` and `grid-template-rows: 48px minmax(0, 1fr) 252px`.
- At `1120x720`: use `grid-template-columns: 272px minmax(420px, 1fr) 272px` and `grid-template-rows: 48px minmax(0, 1fr) 220px`.
- Use 1px region dividers. No region may overlap another region.
- Every region must set `min-width: 0`, `min-height: 0`, and explicit overflow behavior.
- The body remains `min-width: 1120px` and `min-height: 720px`.

## Top Feature Area Contract

Categories:

`媒体`, `音频`, `文字`, `贴纸`, `特效`, `转场`, `字幕`, `滤镜`, `调节`, `模板`, `数字人`

Requirements:

- Replace text-only category buttons with icon-plus-text entries.
- Each entry is 56-64px wide and 40px high; icon area is 18px high, label is 12px.
- Active state is cyan icon/text plus a 2px cyan underline. Do not use a large filled cyan pill.
- Hover state uses a subtle secondary background and does not change button dimensions.
- Unsupported categories remain selectable and show deferred states in the left panel.
- Every category button has `aria-pressed`, an accessible name matching the Chinese label, and a stable size.

## Left Resource Panel Contract

Structure:

- Left panel becomes a two-column internal grid: 56px vertical category tree and `minmax(0, 1fr)` content panel.
- Category tree uses compact 44px rows with icon + 11px label or icon-only with tooltip/aria label if text does not fit.
- Content panel uses a sticky 40px header row, a compact action/search/filter row, then scrollable content.

Media page:

- Header title: `媒体`.
- Primary CTA: `导入素材`.
- Secondary actions: `刷新`, `检查丢失`.
- Search placeholder: `搜索素材`.
- Sort/filter labels: `全部`, `视频`, `图片`, `音频`, `丢失`.
- Material cards are compact rows or two-column mini cards depending on available width.
- Material thumbnail placeholder size is 48x48 in row mode and 96x54 in grid mode.
- Material status badges use success/warning/destructive colors only for availability states.
- Missing and probe-failed materials remain visible with recoverable diagnostics.

Text/audio/subtitle pages:

- Use compact secondary category chips or rows, not large cards.
- Text page keeps current editable text controls and `添加文字`.
- Audio page keeps current BGM selection, duration, volume, track mute, and `添加音频`.
- Subtitle page is visible as deferred unless command semantics already exist.

Deferred pages:

- `贴纸`, `特效`, `转场`, `字幕`, `滤镜`, `调节`, `模板`, `数字人` must remain visible.
- Deferred states use the copy contract above and must not expose disabled blank panels.
- Placeholder controls may show future categories, but they must be visibly non-mutating and cannot call draft mutation code.

## Preview Monitor Contract

Structure:

- Use a professional monitor shell with title bar, black canvas, bottom transport/control bar, and status line.
- Title bar left: draft name, e.g. `未命名草稿`.
- Title bar right: preview state text, e.g. `预览待接入`.
- Canvas: pure black `#070707`, 16:9, centered, with subtle 1px border. The placeholder copy appears centered but small; it must not look like a landing page.
- Bottom controls: icon buttons for play, stop, step backward, step forward, fit, ratio, fullscreen. Real playback remains disabled or shell-only until Phase 05.
- Include a time readout using formatted microseconds from the current playhead display.
- Include reserved slots for future preview frame status and resolution/FPS, but do not compute preview frames.

Interaction:

- Play button can be disabled or shell-only. It must not imply real render playback.
- Stop sets the display playhead to 0 only if it uses existing renderer state; it must not create preview cache or render graph behavior.
- Fit/ratio/fullscreen are UI shell controls only unless existing behavior is implemented.
- All controls must have Chinese `aria-label`s and stable 32px dimensions.

## Inspector Contract

Primary tabs:

`画面`, `音频`, `变速`, `动画`, `调节`, `AI效果`

Structure:

- Top area contains primary tabs in a 32px high compact row.
- Parameter sections use dense rows: label column 72-88px, control column `minmax(0, 1fr)`, optional keyframe button 28px.
- Controls use sliders for continuous values, numeric inputs for exact values, switches/checkboxes for binary settings, segmented controls for modes, and color swatches for colors.
- Every row height is 28-36px. Section headers are 14px semibold.

No selection:

- Show `草稿参数`, not a blank empty state.
- Include read-only or shell controls for draft name, canvas ratio/resolution placeholder, sequence duration, track count, material count, snapping status, and core status.
- Keep command errors visible below the relevant panel area.

Selected video/image segment:

- Show material name, track, source time, target time.
- Show placeholder transform controls: position, scale, rotation, opacity, blend/keyframe shell.
- Placeholder controls are disabled or shell-only unless Rust semantics exist.

Selected text segment:

- Preserve current editable content, font size, color, alignment, stroke, shadow, and background controls.
- `应用文字` must call the existing command helper path and be disabled while a command is pending.
- Color controls use swatches/inputs and must stay compact.

Selected audio segment:

- Preserve volume and track mute behavior through existing command helpers.
- Show waveform/level placeholder only as a visual shell; do not generate waveform data.

Keyframe placeholders:

- Keyframe buttons are visible where appropriate, use a compact diamond/plus symbol or generic icon, and are disabled until command semantics exist.
- Tooltip/aria label: `关键帧功能待接入`.

## Timeline Contract

Structure:

- Timeline region uses three rows: toolbar 40-48px, ruler 28px, scrollable tracks.
- Toolbar is icon-first and compact. Text-heavy action buttons must be replaced with icon buttons plus `title`/`aria-label`.
- Numeric fields for playhead, move, split, and trim may stay visible but must be grouped and compact; they must not force toolbar wrapping at 1120x720.
- Add material control may remain a compact select + `添加片段` button or become an icon action with a visible selected-material selector.
- Include zoom control shell with minus/slider/plus or compact segmented values. Zoom state may be renderer display state only.
- Show snapping status from `workspace.commandState.snapping` as a stable indicator. Do not compute snapping in renderer.

Track headers:

- Fixed width: 144-160px.
- Show track type icon, track name, lock, visibility, mute, and status icons.
- Lock/visibility can be placeholders if command semantics do not exist; mute must preserve existing command behavior for audio/selected tracks.
- Header controls are 24-28px square and never resize on hover/selection.

Ruler/playhead:

- Ruler labels use 12px text and stable tick spacing from existing view model.
- Playhead remains 2px cyan with a 12px handle.
- Moving the playhead through display controls updates renderer display state only.

Segments:

- Video/image segments show thumbnail-like repeated strips using CSS placeholders, not generated thumbnails.
- Audio segments show waveform-like bars using CSS placeholders, not waveform data.
- Text/subtitle segments use red/orange blocks distinct from video/audio.
- Selected segment uses cyan border/outline and does not change size.
- Hover state changes border/background only; no layout shift.
- Segment labels must ellipsize and remain readable at 8px minimum segment width.

Command routing:

- Add/select/move/split/trim/delete/undo/redo must continue to call the existing `Timeline` props and ultimately `window.videoEditorCore.executeCommand`.
- Renderer code must not mutate `draft.tracks`, `track.segments`, source/target timeranges, undo/redo state, or snapping state directly.

## Interaction States

| State | Contract |
|-------|----------|
| Default | Compact dark surfaces, readable Chinese labels, no decorative gradients or floating cards |
| Hover | Border or background changes only; no size, spacing, or text reflow changes |
| Active/selected | Cyan accent on underline, outline, playhead, active segmented option, or focused control |
| Disabled | 48-55% opacity, cursor disabled, same dimensions as enabled state |
| Pending command | Disable mutating controls and show pending text in the timeline/status area |
| Command error | Show destructive-colored Chinese error text near the relevant panel and preserve current layout |
| Empty | Use compact operational empty states with next action; no hero copy |
| Deferred | Visible Chinese not-yet-connected state; no hidden category removal |

Keyboard/accessibility:

- Every icon-only button has `aria-label` and `title`.
- Every category button has `aria-pressed`.
- Selected segment buttons keep `aria-pressed`.
- Inputs keep visible labels; placeholders are not the only label.
- Focus states use the cyan accent border/outline and must be visible on dark backgrounds.

## Boundary And Safety Contract

Renderer may:

- Hold active category, active inspector tab, open panel section, selected display mode, zoom display state, and formatted playhead display state.
- Format integer microseconds for display.
- Render placeholder thumbnails, waveform bars, keyframe buttons, preview controls, and deferred feature states as UI shells.
- Call generated command helpers through the existing Electron boundary.

Renderer must not:

- Construct or mutate canonical draft/timeline semantics directly.
- Assign to or splice `draft.tracks`, `track.segments`, source/target timeranges, undo/redo stacks, or snapping state.
- Construct FFmpeg/ffprobe commands.
- Construct render graphs, filter scripts, export scripts, preview cache keys, thumbnails, waveform data, or proxy paths.
- Store derived artifacts in `.veproj/project.json`.

Source guards must enforce these constraints for Phase 04.1.
