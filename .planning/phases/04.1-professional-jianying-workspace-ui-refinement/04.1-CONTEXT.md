# Phase 04.1: Professional Jianying Workspace UI Refinement - Context

**Gathered:** 2026-06-17
**Status:** Ready for UI design contract and planning
**Mode:** Urgent inserted phase from user request

<domain>
## Phase Boundary

Phase 04.1 upgrades the existing Phase 04 Jianying-style MVP workspace into a denser, more professional desktop editor UI. It focuses on visual structure, information density, panel composition, interaction states, and test-visible command-only behavior before Phase 05 connects real preview/export rendering.

The target is not a brand clone or 100% trade dress copy. The UI should feel like a Jianying Pro-style workstation using original layout, original assets, generic icons, and project-owned implementation.

</domain>

<decisions>
## Implementation Decisions

### Black-Box Reference Use
- The running Jianying Pro app may be used only for black-box observation of reusable design规律: layout hierarchy, panel density, control types, state flow, timeline interaction patterns, typography scale, spacing relationships, and neutral dark color relationships.
- Do not copy Jianying brand assets, proprietary icons, exact artwork, exact presets, private UI resources, or pixel-perfect trade dress.
- Observed usable patterns from the current app window: compact top icon+text category row; cyan active emphasis on a dark UI; left narrow category tree plus content pane; center titled monitor with dark canvas and bottom transport controls; right parameter/inspector pane; bottom timeline with icon toolbar, ruler, playhead, track headers, thumbnail-like video segments, waveform-like audio strips, and colored text/subtitle segment blocks.

### Top Feature Area
- Replace the current text-only top categories with compact icon-plus-text entries.
- Keep Simplified Chinese categories visible: 媒体、音频、文字、贴纸、特效、转场、字幕、滤镜、调节、模板、数字人. Current unsupported categories remain visible as deferred/not-yet-connected states.
- Active state uses restrained cyan accent, not a large filled colored surface.
- Top bar must remain stable and readable at 1120x720 and 1280x800.

### Left Resource And Function Panel
- Convert the left panel into a narrow category tree plus content area.
- Media page includes import, search, sort/filter controls, and compact material grid/card rows with status metadata.
- Text/audio/subtitle pages use secondary categories and compact function cards.
- Sticker/effect/transition/filter/adjustment/template/digital-human pages stay visible with Chinese empty/deferred states.
- Keep material import/check operations routed through existing Rust command helpers.

### Preview Monitor Shell
- Center preview region includes a title bar, black video canvas, and bottom playback controls.
- Playback shell uses familiar icon buttons for play, stop, fit, ratio, fullscreen, and preview status.
- Real preview frame rendering remains Phase 05. Phase 04.1 only reserves the shell and state slots for future preview-frame artifacts.
- Do not construct render graphs, FFmpeg commands, preview cache keys, or export scripts in renderer code.

### Right Inspector
- Inspector uses Jianying-style primary tabs: 画面、音频、变速、动画、调节、AI效果.
- When no segment is selected, show draft/timeline parameters instead of only an empty state.
- When video/image/audio/text segments are selected, show compact controls appropriate to the segment: sliders, numeric inputs, switches, color swatches, segmented controls, and keyframe placeholder buttons.
- Text segment controls must keep existing editable content, font size, color, alignment, stroke, shadow, and background behavior through Rust commands.
- Video/image transform controls can be placeholders if semantics do not exist yet, but they must be clearly visible and non-mutating.

### Timeline
- Replace the current text-heavy toolbar with compact icon buttons and tooltips/aria labels.
- Preserve all current timeline command routing through Rust command APIs: add/select/move/split/trim/delete/undo/redo.
- Track headers show lock, visibility, mute, and type/status icons.
- Segment blocks should look closer to a real editor: video/image segments show thumbnail placeholder strips, audio segments show waveform placeholder bars, text segments use red/orange subtitle-like blocks.
- Ruler, playhead, snapping state, and zoom controls use stable dimensions and must not shift on hover or selection.

### Technical Constraints
- UI emits commands; Rust core owns draft/timeline semantics. Renderer must not directly mutate `draft.tracks`, `track.segments`, segment timeranges, undo/redo stacks, snapping, or MainTrackMagnet.
- Renderer must not construct FFmpeg/ffprobe commands, render graphs, export scripts, preview caches, or waveform generation.
- Persisted semantics still use integer microseconds/rational frame rates. UI may format time for display only.
- All user-visible copy is Simplified Chinese and uses Jianying-style editing terms.
- If adding icons, prefer `lucide-react` generic icons; do not copy Jianying icons.

### Verification
- Add or update Playwright Electron tests for 1280x800 and 1120x720.
- Tests must verify the five major regions are visible and not overlapping/clipped: top feature area, resource panel, preview window, inspector, timeline.
- Tests must verify at least one timeline command still goes through `window.videoEditorCore.executeCommand` and updates UI.
- Source guards must prevent direct renderer draft/timeline mutation, renderer-owned undo/redo/timerange semantics, and renderer FFmpeg/render graph/export/cache construction.
- Final gates should include `just build` and `just test`, or document the exact blocker if they cannot run.

### the agent's Discretion
- Exact component split, CSS class naming, icons, and compact control arrangement are at the agent's discretion if the result meets the professional editor density and verification requirements.
- The agent may defer installing `lucide-react` if compact symbolic controls can be delivered without a new dependency, but icon buttons should be used instead of text-heavy toolbar buttons.

</decisions>
