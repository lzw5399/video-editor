## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Artifact DB -> filesystem deletion | GC decisions turn database rows into filesystem deletes. |
| Artifact DB -> user-visible quota | Byte accounting becomes maintenance UI state. |
| Artifact DB -> local sync manifest | Derived artifact metadata may later be consumed by remote/mobile/server phases. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-14 | Tampering / DoS | `collect_garbage` | mitigate | Use mark-and-sweep from DB rows, validate contained paths, preserve live artifacts, active jobs, source media, DB/WAL/SHM, and project JSON. |
| T-14-15 | Information Disclosure | `compute_quota_state` | mitigate | Return safe labels and aggregate bytes only; do not expose absolute paths, fingerprints, graph keys, dirty ranges, or SQLite internals. |
| T-14-16 | Tampering / Repudiation | `generate_sync_manifest` | mitigate | Deterministic ordering, schema/generator versions, fingerprinted output, and tombstones provide auditable local state without remote side effects. |
| T-14-SC | Tampering | Package installs | accept | No package install in this plan. |
</threat_model>
