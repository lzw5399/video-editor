<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer command payload -> Rust artifact service | Untrusted UI payloads request status, generation actions, quota, and GC. |
| Binding adapter -> artifact_store | Binding layer crosses from JSON transport into local filesystem/SQLite operations. |
| Preview request -> project-derived root | Existing preview transport must stop relying on renderer-owned cache roots. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-17 | Tampering | Generated command contracts | mitigate | Rust-generated schema/TS define allowed payloads and reject mismatched command/payload pairs. |
| T-14-18 | Elevation of Privilege / Tampering | `artifact_store_service.rs` | mitigate | Validate command/session/project inputs and delegate decisions to Rust services; no SQL or cache-key computation in bindings. |
| T-14-19 | Information Disclosure | Status responses | mitigate | Expose safe summaries, display labels, and project-relative diagnostic refs only where allowed; no absolute paths or raw internals. |
| T-14-20 | Tampering | Preview artifact root | mitigate | Rust resolves project-local `.veproj/derived`; renderer `cacheRoot` is optional/deprecated and blocked from default production UI. |
| T-14-SC | Tampering | Package installs | accept | No package install in this plan. |
</threat_model>
