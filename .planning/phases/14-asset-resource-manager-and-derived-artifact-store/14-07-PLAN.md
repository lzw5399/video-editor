<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Generated contracts -> renderer helpers | TypeScript can send commands but must not compute artifact internals. |
| Rust status -> default production UI | Resource status becomes visible and must stay concise, safe, and non-debug. |
| User cleanup action -> Rust GC command | Cache cleanup is destructive for derived blobs and needs confirmation plus Rust-owned candidates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-21 | Tampering | Renderer command helpers | mitigate | Helpers build generated envelopes only; source guards reject artifact-root/key/fingerprint/dirty/GC/SQLite/FFmpeg decisions. |
| T-14-22 | Information Disclosure | Resource status UI | mitigate | UI-SPEC forbidden-copy tests and guards block absolute paths, SQLite names, raw fingerprints, graph keys, dirty ranges, logs, and FFmpeg internals. |
| T-14-23 | Tampering / DoS | Cache cleanup UI | mitigate | Confirmation copy required; renderer sends generated command only; Rust decides cleanup candidates and reports safe result labels. |
| T-14-24 | Repudiation | Final gates | mitigate | `test:phase14` includes Rust behavior tests, source guards, workspace coverage, and contract drift checks; Phase 13 gate remains green. |
| T-14-SC | Tampering | Package installs | accept | No package install in this plan; UI uses existing manual CSS and no icon/component dependency additions. |
</threat_model>
