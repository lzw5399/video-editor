**Adapter-owned compatibility report contract with generated schema and deterministic snapshots for supported, degraded, unsupported, missingResource, and needsNativeEffect diagnostics**

## Accomplishments

- Added the Rust-owned `CompatibilityReport` contract with schema version, source identity, generated timestamp, summary counts, diagnostic items, and optional provenance digest.
- Generated and committed `schemas/compatibility-report.schema.json` from the Rust contract.
- Added deterministic expected report snapshots covering all locked statuses: `supported`, `degraded`, `unsupported`, `missingResource`, and `needsNativeEffect`.
- Added adapter-owned formula-bundle report classification helpers for source material support, degraded text style fallback, unsupported formula blocks, missing resources, and provider-native effects.
- Added tests proving native effects become explicit `needsNativeEffect` report entries and are not hidden as generic filter parameters.

## Decisions Made

- Kept compatibility report types in `adapter_kaipai`, not `draft_model`, because these are external import diagnostics rather than canonical draft semantics.
- Used diagnostic raw formula paths only in `externalPath`; `canonicalTarget` is limited to Jianying-style concepts such as `material` or `text`.
- Omitted canonical targets for unsupported provider blocks and native effects so the report does not imply local semantic support.

- **Fix:** Scoped formula fixture discovery to `fixtures/kaipai/positive` and `fixtures/kaipai/negative`.
