# Phase 03.1 - Validation Strategy

Per-phase validation contract for Kaipai offline formula compatibility foundation.

## Test Infrastructure

| Property | Value |
|----------|-------|
| Framework | Rust `cargo test` with `serde`, `schemars`, `jsonschema`, generated `ts-rs` artifacts; Node/pnpm scripts for source guards |
| Config file | `Cargo.toml`, `package.json`, `justfile` |
| Quick run command | `pnpm run test:phase3_1-fixtures && pnpm run test:phase3_1-source-guards` |
| Full suite command | `just test` |
| Estimated runtime | ~120 seconds |

## Sampling Rate

- After every task commit: run the most specific Rust test for that task plus `pnpm run test:phase3_1-source-guards`.
- After every plan wave: run `pnpm run test` and `just test` when available.
- Before `$gsd-verify-work`: `just test` must pass, generated contracts must be clean, and report snapshots must be intentional.
- Max feedback latency: 120 seconds for targeted checks, 300 seconds for full phase gates.
