<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| formula resource evidence -> local filesystem | Resource paths and hashes from offline bundles drive local copy/reference behavior. |
| localized resources -> `.veproj` project bundle | Renderable resources become project-relative inputs for later preview/export phases. |
| adapter diagnostics -> compatibility report | Missing/unsafe resources become user-visible report items. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-03-CRED | Information Disclosure | resource fixtures | mitigate/blocking | Positive resource fixtures are scanned for tokens, cookies, signed URLs, account IDs, and credentials; negative leakage fixtures must fail validation. |
| T-03.1-03-PATH | Tampering | `ResourceLocalizer` path handling | mitigate/blocking | Tests reject `../`, absolute bundle-relative paths, Windows drive paths for bundle resources, and unsafe output destinations before writes. |
| T-03.1-03-REMOTE | Spoofing / Denial of Service | renderable resource URLs | mitigate/blocking | Remote render URLs cannot become successful localized resources; they emit `missingResource` diagnostics until copied into local relative paths. |
| T-03.1-03-API | Information Disclosure / Elevation of Privilege | localizer implementation | mitigate | Localizer works from offline bundles and filesystem inputs only; no live provider/API, auth, submit/poll, retry, or Android worker runtime calls are allowed. |
| T-03.1-03-RAW | Tampering | project-store/core/render boundaries | mitigate/blocking | Provider fields remain adapter-local; source guards verify `project_store`, core, and render crates do not gain Kaipai/raw formula concepts. |
| T-03.1-03-NATIVE | Repudiation / Integrity | localized native effect resources | mitigate | Localized files do not imply native effect support; native effect behavior remains classified by compatibility reports from Plan 03.1-02. |
| T-03.1-03-SC | Tampering | npm/pip/cargo installs | accept | No new external package installation is planned; use existing `tempfile`, `serde`, and hashing support already available or standard library hashing only if sufficient. |
</threat_model>
