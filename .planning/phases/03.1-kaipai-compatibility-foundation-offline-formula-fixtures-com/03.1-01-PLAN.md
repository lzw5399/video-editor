<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| offline fixture JSON -> `adapter_kaipai` | Untrusted sanitized formula evidence is deserialized into strict Rust contracts. |
| fixture corpus -> git repository | Committed samples can leak provider secrets if sanitizer checks are incomplete. |
| adapter input -> core/render crates | Raw provider concepts must not cross into canonical draft or render semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-01-CRED | Information Disclosure | `fixtures/kaipai` | mitigate/blocking | Sanitizer tests and `rg` checks reject session tokens, signed URLs, cookies, account IDs, production credentials, authorization headers, and token-like keys before fixtures are accepted. |
| T-03.1-01-PATH | Tampering | `FormulaResourceRef.path` | mitigate | Bundle contract accepts resource evidence but path traversal enforcement is completed in Plan 03.1-03; this plan rejects unsafe evidence strings that cannot be represented safely. |
| T-03.1-01-REMOTE | Spoofing / Denial of Service | renderable formula resources | mitigate | Positive fixtures must not contain remote render-time URLs; remote references can appear only as redacted provenance fields and are not canonical draft resources. |
| T-03.1-01-API | Information Disclosure / Elevation of Privilege | adapter crate boundary | mitigate/blocking | No live Kaipai API, auth, submit/poll, provider retry, or Android worker runtime calls are introduced per D-01 and D-07. |
| T-03.1-01-RAW | Tampering | `draft_model`, `engine_core`, `render_graph`, `ffmpeg_compiler` | mitigate/blocking | Raw formula JSON is typed only as adapter input/provenance; later Plan 03.1-04 adds source guards preventing raw/provider terms in core/render crates. |
| T-03.1-01-NATIVE | Repudiation / Integrity | formula evidence | mitigate | Native effect claims are not classified in this plan; Plan 03.1-02 adds explicit `needsNativeEffect` and `unsupported` report snapshots before any mapper claim. |
| T-03.1-01-SC | Tampering | npm/pip/cargo installs | accept | No new external package installation is planned; only an internal workspace crate using existing dependency families is allowed. |
</threat_model>
