# Phase 03.1 Plan 02: Kaipai Offline Formula Fixtures Summary

**Sanitized offline Kaipai formula fixtures with adapter-boundary schema, round-trip, exact-error, and leak-prevention tests**

## Performance

- **Duration:** 10 min

## Accomplishments

- Added two positive offline Kaipai formula bundle fixtures with raw formula evidence, template/recipe/task/request provenance, source media metadata, direct materials, recognizer `word_list`, `safeArea` evidence, and local resource references.
- Added four negative fixtures for missing `word_list`, invalid `safeArea.status`, unsafe `safeArea.source`, and unknown top-level fields.
- Added fixture tests that classify every `fixtures/kaipai` JSON file, validate positives through Rust and generated JSON Schema, round-trip positives deterministically, and assert exact negative errors.
- Added in-memory sanitizer tests for remote URLs, signed URL markers, authorization/cookie/session/access/refresh/account keys, and production-host style remote references without committing unsafe payloads.
- Tightened `KaipaiFormulaBundle` validation so unsafe `safeArea.source`, remote source/direct/resource references, remote formula URLs, signed markers, and credential-like keys fail at the adapter boundary.

## Decisions Made

- Kept unsafe provider data out of committed fixtures entirely; tests generate unsafe URL/token/account/production-host cases in memory.
- Treated `safeArea.source` as adapter provenance only, accepting only redacted local fixture evidence in this phase.
- Kept schema validation and Rust semantic validation distinct: malformed shape fails JSON Schema, while unsafe local provenance can fail only Rust adapter validation.
