## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Formula fixture classifier counted localizer negative fixtures as formula-invalid fixtures**
- **Found during:** Task 2 (local relative path, sha256, and missing-resource tests)
- **Issue:** New localizer negative fixtures are valid formula evidence but invalid localization inputs, so the existing formula negative-fixture test would have treated them as expected formula parse failures.
- **Fix:** Added the positive localizer bundle to formula positives and excluded localizer-specific negative fixtures from formula-contract negative assertions.
- **Files modified:** `crates/adapter_kaipai/tests/fixtures.rs`
- **Verification:** `cargo test -p adapter_kaipai formula_bundle_fixtures -- --nocapture` passed.
- **Committed in:** `a2779f3`

**2. [Rule 3 - Blocking] Fixture guard matched prior report snapshot sourceKind**
- **Found during:** Task 2 verification
- **Issue:** The required fixture guard rejected `kaipaiFormulaBundle` in existing expected report snapshots because it matched the broad `kaipaiFormula` leakage pattern.
- **Fix:** Changed compatibility report `sourceKind` to provider-neutral `offlineFormulaBundle` and regenerated report snapshots through the existing test update path.
- **Files modified:** `crates/adapter_kaipai/src/compatibility_report.rs`, `crates/adapter_kaipai/tests/compatibility_report.rs`, `fixtures/kaipai/expected-reports/*.report.json`
- **Verification:** `cargo test -p adapter_kaipai compatibility_report -- --nocapture` and the fixture source guard passed.
- **Committed in:** `a2779f3`
