<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fixture JSON -> `adapter_kaipai` | Committed offline formula samples are untrusted input to strict Rust contracts. |
| fixture corpus -> git repository | Sample files can leak provider secrets if sanitizer checks are incomplete. |
| raw formula fixture -> canonical draft semantics | Raw provider evidence must not become `.veproj/project.json` render state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-02-CRED | Information Disclosure | `fixtures/kaipai` | mitigate/blocking | Sanitizer tests and `rg` checks reject session tokens, signed URLs, cookies, account IDs, production credentials, authorization headers, and token-like keys before fixtures are accepted. |
| T-03.1-02-PATH | Tampering | fixture resource paths | mitigate | Positive fixtures use local-relative or redacted resource evidence; Plan 03.1-04 validates path traversal and localizer behavior for renderable resources. |
| T-03.1-02-REMOTE | Spoofing / Denial of Service | fixture resource references | mitigate | Positive fixtures must not contain remote render-time URLs; remote references can appear only as in-memory negative sanitizer cases. |
| T-03.1-02-API | Information Disclosure / Elevation of Privilege | fixture generation | mitigate/blocking | No live Kaipai API, auth, submit/poll, provider retry, or Android worker runtime calls are introduced per D-01 and D-07. |
| T-03.1-02-RAW | Tampering | expected `.veproj/project.json` artifacts | mitigate/blocking | Raw formula JSON is committed only as adapter formula fixture evidence; source guards in Plan 03.1-05 reject raw formula leakage into canonical draft artifacts. |
| T-03.1-02-SC | Tampering | npm/pip/cargo installs | accept | No new external package installation is planned; fixture tests reuse existing Rust dependency families. |
</threat_model>
