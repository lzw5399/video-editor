<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| formula bundle evidence -> report generator | Untrusted adapter input is converted into user-facing compatibility diagnostics. |
| report snapshots -> git repository | Report fixtures can leak provider identifiers or normalize unsafe evidence if unchecked. |
| native provider effect -> canonical draft target | Proprietary native behavior must not be falsely represented as supported draft semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-02-CRED | Information Disclosure | `fixtures/kaipai/expected-reports` | mitigate/blocking | Snapshot tests and Phase 03.1 source guards reject signed URLs, cookies, account IDs, tokens, and credentials in reports. |
| T-03.1-02-PATH | Tampering | report `externalPath` / resource entries | mitigate | Report paths are diagnostic strings only; Plan 03.1-03 enforces path traversal rejection for localized renderable resources. |
| T-03.1-02-REMOTE | Spoofing / Denial of Service | missing resource diagnostics | mitigate | Missing or remote resources become `missingResource` report items; reports do not instruct preview/export to fetch remote template URLs. |
| T-03.1-02-API | Information Disclosure / Elevation of Privilege | report generation | mitigate/blocking | Report generation must not call live Kaipai APIs, submit/poll endpoints, auth flows, provider retries, or Android worker runtime code. |
| T-03.1-02-RAW | Tampering | report canonical targets | mitigate | Raw formula paths may appear only as external diagnostic paths; canonical targets must use Jianying-style draft/material/track/segment/keyframe/filter/transition names. |
| T-03.1-02-NATIVE | Repudiation / Integrity | `CompatibilityStatus::NeedsNativeEffect` | mitigate/blocking | Tests require `needsNativeEffect` snapshots and guard against native effect smuggling into `Filter.parameters`. |
| T-03.1-02-SC | Tampering | npm/pip/cargo installs | accept | No new external package installation is planned; continue using existing Rust dependency families. |
</threat_model>
