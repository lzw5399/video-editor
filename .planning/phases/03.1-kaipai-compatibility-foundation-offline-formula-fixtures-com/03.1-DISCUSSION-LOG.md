## Offline Formula Boundary

| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** Auto-selected the spike-validated offline formula bundle path.
**Notes:** This preserves the user's intent to avoid depending on Android worker and avoid live Kaipai API complexity in the first compatibility phase.

## Layer Ownership

| Option | Description | Selected |
|--------|-------------|----------|
| Adapter/service boundary | Keep formula parsing, provenance, resource localization, and report generation outside core/render crates | yes |
| Core understands raw formula | Let `draft_model` or `engine_core` interpret Kaipai JSON directly | |

**User's choice:** Auto-selected strict adapter/service boundary.
**Notes:** This follows project constraints: UI emits commands, Rust core owns semantics, render graph and FFmpeg compiler do not interpret provider JSON.
