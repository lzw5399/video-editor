<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer JSON -> `CommandEnvelope` | Untrusted command payloads deserialize into strict Rust command variants. |
| Binding route -> `draft_commands` | Native binding must delegate edit semantics to the pure command crate. |
| Command transaction -> returned draft/state | Add/move/split/trim/delete must commit only after validation succeeds. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Malformed timeline edit payloads | mitigate | Strict typed payloads, command/payload matching, schema fixtures, and binding invalid-payload tests. |
| T-03-02 | Tampering/DoS | Timerange arithmetic in split/trim/move | mitigate | Use Plan 03-01 checked microsecond helpers for all source/target updates. |
| T-03-03 | Tampering | Locked-track and overlap mutation | mitigate | Call locked-track, track/material compatibility, material-bound, and no-overlap checks before commit. |
| T-03-04 | Repudiation/Tampering | Rejected edit history mutation | mitigate | Clone draft/state/selection and push no history on rejection; tests assert unchanged state. |
| T-03-05 | Elevation of Privilege/Tampering | Renderer bypass of Rust semantics | mitigate | Binding exposes generated commands only and delegates all timeline edit behavior to `draft_commands`. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install is planned. Stop for human legitimacy verification if a new dependency is proposed. |
</threat_model>
