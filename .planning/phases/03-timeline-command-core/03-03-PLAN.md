## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Command response -> Electron state | Rust returns updated draft, selection, session history, and events to the renderer. |
| Edit transaction -> history stack | Only committed edits may alter undo/redo state. |
| Move/trim payload -> snapping engine | Untrusted target times and snap settings are normalized by Rust-owned snapping logic. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Malformed undo/redo or snapping payloads | mitigate | Strict generated payloads and binding invalid-payload tests. |
| T-03-02 | Tampering/DoS | Snapping and magnet timerange math | mitigate | Use checked microsecond helpers and deterministic thresholds; no float seconds. |
| T-03-03 | Tampering | Locked-track edits after snap/magnet adjustment | mitigate | Re-run Plan 03-01 validation after snapping/magnet modifications and before commit. |
| T-03-04 | Repudiation/Tampering | Rejected-command history mutation | mitigate | Push undo snapshots only after validated commits; rejected commands do not clear redo. |
| T-03-05 | Elevation of Privilege/Tampering | Renderer-owned undo/snap interpretation | mitigate | Renderer receives opaque/generated state and events; binding delegates undo/snap/magnet behavior to `draft_commands`. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install is planned. Stop for human legitimacy verification if a new dependency is proposed. |
</threat_model>
