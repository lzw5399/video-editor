<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| media path -> project_store | User-selected local file paths/URIs enter path resolution and draft material registry updates. |
| project_store -> media_runtime | Resolved local media path is probed by bounded ffprobe execution. |
| ffprobe JSON -> media_runtime | External process output is parsed into normalized metadata. |
| Rust binding -> Electron renderer | Generated material metadata crosses into smoke UI display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering/Information Disclosure | Material URI resolution | mitigate | Resolve relative/external material URIs only in `project_store`; reject traversal/ambiguous bundle paths and preserve original URI for diagnostics. |
| T-02-11 | Denial of Service | ffprobe process | mitigate | Use `FfmpegExecutor` argument arrays, existing timeout, bounded stdout/stderr summaries, and no shell strings. |
| T-02-12 | Tampering | ffprobe JSON parser | mitigate | Parse into normalized typed metadata only; reject malformed JSON with classified probe errors and never persist raw probe JSON. |
| T-02-13 | Elevation of Privilege | Renderer-native bridge | mitigate | Renderer uses generated preload command API only; no renderer FFmpeg/process/filesystem access. |
| T-02-14 | Tampering | Missing material handling | mitigate | Missing files produce recoverable diagnostics and preserve material entries exactly per D-14/D-15. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package-manager install is planned; existing serde_json is already approved in workspace. Stop for human legitimacy verification if a new dependency is proposed. |
</threat_model>
