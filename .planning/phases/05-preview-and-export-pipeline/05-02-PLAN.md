<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| engine_core to render_graph | Pure semantic frame/range state becomes renderer-neutral intent data. |
| render_graph to ffmpeg_compiler | Renderer-neutral graph becomes FFmpeg-specific plans and derived sidecars. |
| ffmpeg_compiler to media_runtime | Structured argument vectors and sidecar contents become executable runtime inputs in later plans. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering | `render_graph` builder | mitigate | Build only from `engine_core` normalized range state; reject renderer-owned layer lists and unsupported graph invariants with classified errors. |
| T-05-06 | Elevation of Privilege | FFmpeg args | mitigate | Emit `Vec<OsString>` argument plans and filter scripts; never shell-concatenate user or renderer input. |
| T-05-07 | Information Disclosure | sidecar contents | mitigate | Treat ASS/filter scripts as derived artifacts outside `.veproj/project.json`; snapshot outputs use fixture paths only. |
| T-05-08 | Denial of Service | compiler profile handling | mitigate | Validate output dimensions, target timeranges, profile kind, and required labels before runtime execution. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new external package installs; use existing approved dependencies and local crates only. |
</threat_model>
