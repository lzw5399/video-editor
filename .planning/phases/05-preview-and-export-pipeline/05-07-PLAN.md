<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compiler/service to media_runtime | Structured FFmpeg args cross into process execution. |
| media_runtime to filesystem | FFmpeg writes MP4 output and ffprobe reads it for validation. |
| runtime to binding/UI | Progress, cancel, logs, and errors cross as structured job status. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-13 | Tampering | FFmpeg process args | mitigate | Runtime consumes explicit `Vec<OsString>` args and never shell-concatenates renderer/user input. |
| T-05-14 | Denial of Service | export job runtime | mitigate | Progress parsing, timeout, cancel token, and bounded log summaries prevent hung or unbounded jobs. |
| T-05-15 | Information Disclosure | export logs | mitigate | Return bounded log summaries and classified errors; do not persist logs in `.veproj/project.json`. |
| T-05-16 | Tampering | output validation | mitigate | Validate rendered output with ffprobe metadata for file existence, duration, fps, resolution, audio stream, and non-empty size. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new external package installs; only local crates and already-approved workspace dependencies may be wired. |
</threat_model>
