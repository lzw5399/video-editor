<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer to export commands | User-selected output path/preset crosses into Rust-owned validation and job planning. |
| ffmpeg_compiler to media_runtime | Structured job plan crosses into process execution. |
| media_runtime to filesystem | FFmpeg writes exported MP4 and ffprobe validates output metadata. |
| testkit to runtime tools | Golden parity tests invoke local FFmpeg/ffprobe through runtime boundaries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-13 | Tampering | FFmpeg process args | mitigate | Runtime consumes explicit `Vec<OsString>` args and never shell-concatenates renderer/user input. |
| T-05-14 | Denial of Service | export job runtime | mitigate | Progress parsing, timeout, cancel token, and bounded log summaries prevent hung or unbounded jobs. |
| T-05-15 | Information Disclosure | export logs/UI | mitigate | Return bounded log summaries and classified errors; do not persist logs in `.veproj/project.json`. |
| T-05-16 | Tampering | output validation | mitigate | Validate rendered output with ffprobe metadata for file existence, duration, fps, resolution, audio stream, and non-empty size. |
| T-05-17 | Elevation of Privilege | renderer export UI | mitigate | Source guards block renderer-owned process, FFmpeg, render graph, export script, cache, and validation construction. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new external packages; final source guards and lockfile discipline preserve the approved dependency set. |
</threat_model>
