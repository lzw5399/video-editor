<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| binding/service to preview_service | Preview requests enter Rust-owned cache/render orchestration. |
| preview_service to filesystem cache | Derived preview artifacts are written outside canonical draft state. |
| accepted edit result to cache invalidation | Semantic edits become invalidation ranges and material dependencies. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08 | Tampering | preview cache | mitigate | Cache entries include target range, profile, semantic fingerprint, and material dependencies; invalidation is range-based and conservative. |
| T-05-09 | Information Disclosure | cache artifacts | mitigate | Write artifacts only under configured cache roots and return bounded artifact metadata, not raw script/log internals. |
| T-05-10 | Denial of Service | preview generation | mitigate | Use media_runtime execution boundaries and classified errors so failed renders preserve draft state. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new external package installs; only local crates and already-approved workspace dependencies may be wired. |
</threat_model>
