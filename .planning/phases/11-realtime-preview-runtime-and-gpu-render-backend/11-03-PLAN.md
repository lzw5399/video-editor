<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| material metadata -> frame provider | Draft material IDs and source positions select cached frame data. |
| decoded frame cache -> runtime | Session-owned decoded CPU frames cross into runtime rendering code. |
| fixture generation -> preview request | Test fixture generation may use FFmpeg, but realtime requests must not. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-07 | Denial of service | CPU frame validation | mitigate | Validate dimensions, stride, pixel length, source position, and cache bounds before returning frames. |
| T-11-08 | Tampering | frame selection | mitigate | Select cached frames by integer microseconds/frame index; no floating-point persisted time or UI-owned frame selection. |
| T-11-09 | Repudiation | realtime video path | mitigate | Tests prove `frame_for` does not invoke FFmpeg per frame and diagnostics identify uncached/unavailable frames. |
| T-11-SC | Tampering | package installs | mitigate | This plan adds no external package beyond previously approved Phase 11 runtime dependencies. |
</threat_model>
