<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| binding/main -> realtime runtime | Untrusted UI-triggered requests enter Rust session APIs. |
| runtime request -> session clock | In-flight results can be stale after seek/edit/pause/resume. |
| package install -> workspace | New Rust crate dependencies must match the Package Legitimacy Audit in `11-RESEARCH.md`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | `RealtimePreviewFrameResult` presentation | mitigate | Reject presentation when `request.playback_generation != session.clock.generation` and test stale rejection. |
| T-11-02 | Spoofing | `PreviewSessionId` | mitigate | Keep session IDs opaque `u64` values allocated by Rust; do not accept renderer-created GPU/device handles. |
| T-11-03 | Denial of service | generation counters and request telemetry | mitigate | Use bounded in-memory counters and explicit session close/reset APIs; no unbounded per-request log vector in the clock layer. |
| T-11-SC | Tampering | cargo dependencies | mitigate | Use only packages marked OK in `11-RESEARCH.md` Package Legitimacy Audit; no `[ASSUMED]`, `[SUS]`, or `[SLOP]` install proceeds in this plan. |
</threat_model>
