# Phase 15.1 Plan 06: Full User-Chain Verification Summary

Phase 15.1 is closed with a hard user-chain gate: a normal desktop user can import repo-owned media, add it to the timeline, play through the realtime host without a `requestPreviewFrame` playback loop, observe changing visible composited output, run preview/export workflows in dev and packaged apps, and pass no-product-fallback guards.

## Accomplishments

- Added a macOS native composited preview presenter in `bindings_node` that attaches an `AVPlayerLayer` child view to the Electron window for the baseline single local-video path.
- Added presentation evidence that samples the actual composited child-view screen region and records changing content digests into realtime telemetry.
- Wired Electron main/preload/renderer state so playback uses the realtime preview host and does not treat preview-frame artifacts, mock output, decoded CPU frames, or FFmpeg artifacts as product playback success.
- Preserved fail-closed behavior: unsupported presenter paths, missing native surfaces, multi-segment/effects/text/keyframe-heavy unsupported compositions, and unavailable production presenters return explicit unavailable diagnostics instead of silently substituting fallback output.
- Fixed sequential material import state so a real multi-file import does not overwrite the first accepted material with stale React state.
- Added repo-owned generated media fixture expectations for real workflow export validation.
- Hardened Playwright product and real-workflow tests:
  - product playback requires backend `gpu`, composited evidence, changing content digest, advancing target time, changing visible preview pixels, and no playback-time `requestPreviewFrame` calls;
  - real workflow runs no-mock dev and packaged import-preview-export with real video/audio fixtures.
- Updated workspace tests and layout CSS for the no-fallback semantics and compact timeline controls.
- Updated runtime-boundary docs with the no-product-fallback product policy.

## Important Scope Boundary

This plan connects the baseline single-video native/composited product playback path and proves it end to end. It does not claim that the final multi-track/effects/text/render-graph GPU compositor is complete. Unsupported realtime compositions must remain explicit unavailable states until later production compositor work supports them.

## Verification

See `15.1-VERIFICATION.md` for the full command log. All Phase 15.1 plan gates passed, including product playback E2E, dev and packaged real workflow, workspace suite, Rust runtime/binding tests, source guards, no-product-fallback guard, full workspace check, and `git diff --check`.

## Deviations

- `workspace.spec.ts` no longer treats mock realtime backend success as product playback success. Workspace mock/display tests were aligned with the project no-product-fallback policy; product success is proven by `product-user-journey.spec.ts`.
- The `ffmpeg_compiler text` snapshot was updated to match the current explicit canvas-background composition model already used by transform snapshots.
