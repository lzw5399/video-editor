# Phase 15.1 Plan 05: Multitrack Commands Summary

Video, audio, and text tracks can now be created, selected as timeline targets, renamed, locked, shown/hidden, and muted through Rust-owned commands and generated desktop contracts.

## Accomplishments

- Added `Track.visible` with backward-compatible Rust serde defaults and regenerated JSON schema/TypeScript contracts.
- Added `addTrack`, `renameTrack`, `setTrackLock`, and `setTrackVisibility` command payloads, binding routing, undo/redo snapshots, command events, and targeted track deltas.
- Extended dirty-domain propagation with `DirtyDomain::Track`; visual track visibility invalidates preview/export/thumbnail/cache consumers for affected segments.
- Updated `engine_core` visual stack ordering and normalization to skip hidden visual tracks.
- Wired desktop timeline controls for adding video/audio/text tracks, selecting a target track, renaming, lock/unlock, visual show/hide, and audio mute.
- Updated add-segment UI behavior so the selected compatible track is used before falling back to the first compatible track.
- Added Playwright coverage for the real user flow across add/select/add segment/rename/lock/visibility/mute.
- Strengthened the project no-product-fallback policy as a mandatory review gate.
