<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| packaged renderer -> preload -> Electron main | Untrusted renderer can only access the typed preload bridge. |
| Electron main -> native Node-API binding | Main process resolves a native `.node` wrapper from dev or packaged resource paths. |
| Rust binding -> external FFmpeg runtime | Runtime binaries come from env/PATH and are probed through Rust-owned commands. |
| npm registry -> dev dependency install | `electron-builder` enters the build toolchain. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege | `apps/desktop-electron/src/main/index.ts` IPC | mitigate | Preserve `assertAllowedIpcSender`, context isolation, sandbox, and packaged `file:` allowlist; packaged smoke must verify bridge shape and reject untrusted navigation inheritance. |
| T-06-02 | Tampering | `nativeBinding.ts` packaged path resolution | mitigate | Resolve deterministic packaged candidates under `process.resourcesPath`/`app.asar.unpacked`, validate API shape before caching, and keep `VE_NATIVE_BINDING_PATH` as an explicit test override only. |
| T-06-03 | Denial of Service | packaged runtime probe | mitigate | Use existing Rust bounded process runtime and assert invalid FFmpeg paths return classified envelope errors without crashing the app. |
| T-06-SC | Tampering | `electron-builder` install | mitigate | Use audited `electron-builder@26.15.3` from the Phase 06 Package Legitimacy Audit; update `pnpm-lock.yaml`; do not add unaudited packaging dependencies. |
</threat_model>
