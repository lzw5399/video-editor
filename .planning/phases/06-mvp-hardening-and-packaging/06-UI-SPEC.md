# Phase 06 - UI Design Contract

> Visual and interaction contract for Phase 06: MVP Hardening And Packaging. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 06 is a hardening and packaging phase. It must not introduce a marketing page, onboarding flow, new primary navigation, or new editor semantics.

The UI work for this phase is limited to runtime/package diagnostics, preview/export capability states, packaged-app error states, and preservation of the existing professional Jianying-style workspace density. All user-facing desktop copy must be Simplified Chinese. Do not copy CapCut/Jianying branded assets, icons, colors, or trade dress; keep the project-owned open-source UI while using familiar editor terminology.

Sources used: `.planning/REQUIREMENTS.md` UI-01 through UI-12, TEST-06 through TEST-08; `06-CONTEXT.md` D-03, D-08, D-09, D-11, D-16; `apps/desktop-electron/src/renderer/styles.css`; `apps/desktop-electron/tests/workspace.spec.ts`.

## Copywriting Contract

All copy below is locked for Phase 06 unless implementation discovers a command-specific Rust error category that requires a more precise Chinese label.

| Element | Copy |
|---------|------|
| Primary CTA | 检查运行环境 |
| Secondary CTA | 重新检测 |
| Preview disabled CTA state | 预览暂不可用 |
| Export disabled CTA state | 导出暂不可用 |
| Empty state heading | 等待运行环境检测 |
| Empty state body | 打包应用启动后会检测剪辑核心、FFmpeg、编码器和字幕能力。 |
| Ready state | 运行环境就绪 |
| Packaged launch ready | 打包应用已就绪 |
| Capability warning | 部分能力不可用，可继续编辑，但预览或导出可能受限。 |
| Error state | 运行环境检测失败，请检查 FFmpeg/ffprobe 路径后重试。 |
| Missing FFmpeg | 未找到 FFmpeg，请配置 VE_FFMPEG_PATH 或加入 PATH。 |
| Missing ffprobe | 未找到 ffprobe，请配置 VE_FFPROBE_PATH 或加入 PATH。 |
| Binding error | 剪辑核心加载失败，请重新安装或使用开发构建验证。 |
| Encoder unavailable | 当前 FFmpeg 不支持 H.264/AAC 导出，请更换可用构建。 |
| Subtitle filter unavailable | 当前 FFmpeg 缺少字幕滤镜，文字预览和导出可能受限。 |
| Font readiness warning | 字体环境未完全就绪，文字渲染可能与导出结果不一致。 |
| External runtime warning | 当前使用本机 FFmpeg，仅用于本地测试，不代表可再发行构建。 |
| Export cancellation | 取消导出：导出任务将停止，已生成的临时文件可能会被保留。 |

Diagnostic wording must be actionable: state what failed, what capability is affected, and the next step. Do not expose raw Rust enum names, internal file paths as primary copy, FFmpeg command lines, render graph details, or English-only package errors.

## Phase 06 Interaction Contract

Runtime diagnostics must live inside the existing editor workspace. Preferred placement is the current preview/export area or an existing compact diagnostic list in the material/preview shell. Do not add a new route, splash screen, marketing page, side rail, modal-first workflow, or duplicate left primary menu.

Primary focal point remains the preview monitor. Runtime diagnostics are secondary support information and must not visually compete with the preview canvas, playback controls, export controls, or timeline editing controls.

Required capability states:

| State | Visual Treatment | Interaction Rules |
|-------|------------------|-------------------|
| Checking | Compact secondary panel, spinner/progress text if available, text `正在检测运行环境` | Disable preview frame, preview segment, and export start buttons; editing workspace stays visible. |
| Ready | Success dot/badge with `运行环境就绪` | Enable preview/export actions according to existing command availability. |
| Warning | Amber diagnostic list using existing warning color | Keep editing available; disable only the affected preview/export action when the missing capability blocks it. |
| Error | Red error text inside existing panel structure | Do not crash or blank the renderer; disable affected actions and show `重新检测`. |
| Packaged offline | Same as Ready or Warning after launch from packaged artifact | Must verify preload bridge, binding, ping/version, runtime probe, and file-based renderer load without network UI. |

Preview/export actions must remain command-owned:

- Preview buttons call generated command helpers through `window.videoEditorCore.executeCommand`; no renderer FFmpeg probing or cache invalidation logic.
- Export start/status/cancel calls remain generated command helpers; no renderer render graph, FFmpeg args, process handles, validation expectations, or script construction.
- Capability reports displayed in UI are read-only summaries returned from Rust/binding-owned runtime checks.
- If runtime capability is unknown, show disabled action labels rather than optimistic enabled buttons.

Diagnostics content:

- Show discovered `ffmpeg` and `ffprobe` status as `可用` / `未找到` / `不可用`.
- Show version/configure summary as one truncated line with a full value available in accessible text or a title attribute.
- Show encoder readiness for `H.264` and `AAC`.
- Show subtitle/filter readiness for ASS/subtitles rendering.
- Show deterministic font readiness.
- Show external-runtime/license posture as a warning when using Homebrew or PATH-provided FFmpeg.

No diagnostic state may resize the five workspace regions. Status text may scroll inside its panel or truncate with ellipsis, but it must not push the preview canvas, inspector, or timeline out of bounds.

## Viewport And Layout Contract

The current professional workspace proportions are locked for Phase 06.

| Viewport | Expected Layout |
|----------|-----------------|
| 1280x800 | Top feature bar 48px high; left panel 336px; inspector 288px; timeline 252px; center preview gets remaining width with min 420px; all five regions visible and non-overlapping. |
| 1120x720 | Minimum supported desktop viewport; left panel 272px; inspector 272px; timeline 220px; top feature bar 48px; timeline status/playhead time may hide; all controls remain inside their strips. |

Required layout gates:

- Main region remains `剪映风格编辑工作区`.
- Regions with labels `顶部功能区`, `素材面板`, `预览窗口`, `属性检查器`, and `时间线` are visible at both viewports.
- Top category labels do not wrap or overflow.
- No `.secondary-category-rail`, `.secondary-category-button`, or duplicate left primary navigation appears.
- Preview canvas remains 16:9 with ratio tolerance <= 0.04.
- Preview controls, artifact lines, export controls, progress, logs, validation, and Phase 06 diagnostics stay inside `.preview-shell`.
- Timeline controls stay inside `[aria-label="时间线控制"]`.
- Hover, selection, checking, ready, warning, error, export progress, export complete, and export canceled states do not change preview/timeline/inspector region box sizes by more than 1px.
- Global scrollbars remain compact dark scrollbars: `scrollbar-width: thin` and WebKit scrollbar width/height `4px`.
