<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| media file metadata -> media_runtime contracts | Untrusted container/stream facts become typed requests, stream info, and decoded-frame metadata. |
| runtime session -> frame/texture handles | Opaque IDs can be retained, released, or rejected based on owner session and generation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01-01 | Tampering | `FrameHandleId` / `TextureHandleId` | mitigate | Store owner session and generation on every handle and test mismatch rejection in the frame-pool contract. |
| T-12-01-02 | Denial of Service | `FramePool` | mitigate | Add bounded outstanding lease counters and session-close leak diagnostics before any platform decoder can allocate native resources. |
| T-12-01-03 | Information Disclosure | texture/native handle contracts | mitigate | Expose opaque IDs and metadata only; raw native pointers and full frame bytes are not public contract fields. |
| T-12-01-SC | Tampering | npm/pip/cargo installs | accept | This plan adds no package-manager dependencies. Package verification is handled in 12-02B before platform dependency additions. |
</threat_model>
