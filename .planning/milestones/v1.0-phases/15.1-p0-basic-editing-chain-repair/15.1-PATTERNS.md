# Phase 15.1 Patterns

**Date:** 2026-06-19
**Status:** Ready for execution planning

## Existing Patterns To Reuse

### Rust-owned command semantics

- `crates/draft_commands/src/timeline.rs` is the pattern for accepted timeline mutations, validation, selection updates, undo snapshots, and generated `TimelineCommandResponse`.
- `crates/draft_commands/src/canvas.rs` is the pattern for draft canvas mutation.
- `crates/draft_commands/src/audio.rs` is the pattern for track mute and audio-specific command validation.
- `crates/draft_commands/src/delta.rs` is the pattern for invalidation facts; renderer should not invent semantic dirty domains.

### Generated contracts

- `crates/draft_model/tests/schema_exports.rs` is the source of truth for generated schema/TypeScript updates.
- `schemas/command.schema.json`, `schemas/draft.schema.json`, `apps/desktop-electron/src/generated/*` are generated artifacts. Update through existing test/generation path when command/model contracts change.

### Realtime preview

- `crates/realtime_preview_runtime/src/session.rs` owns session lifecycle, `TimelineClock`, generation, surface attach, frame requests, and telemetry.
- `crates/bindings_node/src/realtime_preview_service.rs` maps runtime sessions to binding IDs and validates realtime payloads.
- `apps/desktop-electron/src/main/realtimePreviewHost.ts` is the desktop host bridge for native/offscreen preview surface ownership.
- `apps/desktop-electron/src/preload/index.ts` exposes narrow host bridges to renderer without giving renderer Electron access.

### Desktop command boundary

- `apps/desktop-electron/src/renderer/commandHelpers.ts` constructs typed command envelopes.
- `App.tsx` applies command results through `applyTimelineCommandResult`, not by mutating canonical draft semantics directly.
- Source guards from earlier phases should be extended rather than replaced, especially `scripts/phase15-source-guards.sh`.

### Electron E2E

- `apps/desktop-electron/tests/workspace.spec.ts` already records command calls via `VIDEO_EDITOR_TEST_RECORD_COMMANDS`.
- `apps/desktop-electron/tests/real-workflow.spec.ts` and `tests/helpers/realWorkflow.ts` are the closest no-mock user-chain patterns.
- Test-only main-process command responses in `apps/desktop-electron/src/main/index.ts` are acceptable for UI/E2E tests when explicitly gated by env vars.

## Anti-Patterns To Avoid

- Do not make playback "work" by enabling a disabled button while keeping the UI-owned `requestAnimationFrame` plus `requestPreviewFrame` loop.
- Do not use FFmpeg preview artifacts as normal realtime playback success for supported baseline media.
- Do not resolve text fonts through host-specific paths without a bundled registry and license manifest.
- Do not infer canvas orientation in renderer-only code.
- Do not mutate `draft.tracks` or segment placement directly in React when adding multitrack controls.
- Do not move scheduler, retiming, advanced effects, masks, transitions, or Phase 15.2 UI convergence into Phase 15.1.

## File Ownership Map

- Playback runtime and binding: `crates/realtime_preview_runtime/src/*`, `crates/bindings_node/src/realtime_preview_service.rs`, `crates/bindings_node/src/lib.rs`, `apps/desktop-electron/src/main/nativeBinding.ts`, `apps/desktop-electron/src/main/realtimePreviewHost.ts`, `apps/desktop-electron/src/preload/index.ts`, `apps/desktop-electron/src/renderer/App.tsx`, `PreviewMonitor.tsx`, `Timeline.tsx`.
- Font registry: `crates/draft_model`, `crates/media_runtime`, `crates/ffmpeg_compiler`, `crates/realtime_preview_runtime`, `crates/artifact_store`, `apps/desktop-electron/src/renderer/*`, `assets/fonts` or a new product asset directory.
- Canvas adaptation: `crates/draft_commands/src/canvas.rs`, `crates/draft_commands/src/timeline.rs`, `crates/bindings_node/src/lib.rs`, command generated files, `App.tsx` only as caller.
- Multitrack commands: `crates/draft_model/src/timeline.rs`, `crates/draft_commands/src/timeline.rs`, `delta.rs`, generated command contracts, `Timeline.tsx`, inspector/view model.
- User-chain acceptance: `apps/desktop-electron/tests/*`, `scripts/phase15-1-source-guards.sh`, `.planning/phases/15.1-*/15.1-VERIFICATION.md`.
