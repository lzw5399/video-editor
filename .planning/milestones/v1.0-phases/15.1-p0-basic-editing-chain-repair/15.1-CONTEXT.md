# Phase 15.1: P0 Basic Editing Chain Repair - Context

**Gathered:** 2026-06-19
**Status:** Ready for research and planning
**Source:** User PRD input plus ROADMAP insertion

<domain>

## Phase Boundary

Phase 15.1 repairs the default editing chain before scheduler work starts. The target is a production editor baseline, not a demo fallback path:

- Playback must be real realtime preview playback for supported baseline media, not `requestAnimationFrame` driving repeated `requestPreviewFrame` PNG refreshes.
- Baseline video/image/text/audio preview must either use supported production paths or fail clearly; FFmpeg preview artifact fallback is not normal playback success.
- Text preview/export must use the same bundled, license-tracked font registry.
- Empty drafts must adapt canvas orientation/aspect/frame rate from the first video/image material unless the user has manually changed canvas settings.
- Multitrack editing must be explicit enough for users to add/target/rename/lock/show/mute tracks and stack/mix segments through Rust commands.
- A real end-to-end fixture must prove import, edit, play, text, audio, trim, split, undo/redo, save, reopen, and export.

This phase may touch Rust core, bindings, Electron renderer, fixtures, and tests. It must not advance scheduler policy from Phase 16 or the UI convergence work reserved for Phase 15.2 except where a minimal UI surface is required to prove the editing chain.

</domain>

<decisions>

## Locked Decisions

### Playback Chain

- Normal playback must not continuously call `requestPreviewFrame`.
- `requestPreviewFrame` is allowed for paused single-frame preview, explicit scrub/seek still-frame refresh, or developer diagnostics.
- Play/pause/seek/scrub must route through the formal realtime preview runtime and shared integer microsecond `TimelineClock` / `PlaybackGeneration` model.
- Supported baseline playback must not silently pass through FFmpeg PNG/preview segment fallback.

### Baseline Supported Path

- Supported baseline means ordinary video/image display, baseline text overlay, and audio playback/mix for production preview.
- If a baseline capability is unavailable, the app must surface productized failure instead of pretending fallback output is equivalent.
- Existing advanced retiming/effects/filters/masks/transitions remain Phase 18 scope.

### Font Registry

- Bundled open-source Chinese-capable font assets must be tracked through a registry with stable `fontRef`, family, weight/style, file path, and license metadata.
- Realtime preview, FFmpeg/ASS export, and UI font selection must resolve font data through the same registry.
- Missing font files, missing license manifest entries, and missing glyph coverage must be explicit errors.

### First-Material Canvas Adaptation

- Empty drafts use the first video/image material metadata to choose canvas dimensions/aspect and frame rate.
- Portrait media should produce portrait canvas, square media square canvas, and landscape media landscape canvas.
- If the user manually changed canvas settings, later material additions must not overwrite them.
- Default segment visual fit must avoid stretch distortion.

### Multitrack Editing

- Users must be able to create and target video/audio/text tracks explicitly.
- Video/image/text layers stack deterministically; audio tracks mix deterministically.
- Track rename, lock, show/hide, and mute must be Rust-owned commands with generated contracts and undo/redo behavior where applicable.
- Renderer code must not mutate draft tracks/segments directly.

### User-Chain Acceptance

- Phase completion requires an executable E2E flow with real fixture media: import vertical video, auto vertical canvas, play, add bundled-font text, add music, trim, split, undo/redo, save, reopen, export.
- The E2E must assert no continuous `requestPreviewFrame` calls during playback.

### the agent's Discretion

- Choose exact plan/task split, fixture names, and whether canvas adaptation is implemented as command logic, material import logic, or add-to-timeline logic, as long as Rust owns the semantic decision.
- Choose the bundled font package after verifying license compatibility.
- Choose the minimal production UI controls needed to expose multitrack and font registry functionality; broad visual cleanup is Phase 15.2.

</decisions>

## Specific Ideas

- Add a regression test that starts playback, records command calls, and asserts `requestPreviewFrame` does not repeat as the playback clock advances.
- Add first-material portrait fixture metadata and a test proving canvas/profile update before segment insertion.
- Add track command tests before UI work: add track, rename track, set lock, set visibility, set mute, target add segment.
- Add font registry tests before UI/export integration: bundled font manifest, license metadata, `fontRef` resolution, ASS export using registry file.
- Add a full Electron E2E fixture flow and keep it as the Phase 15.1 aggregate gate.

</specifics>
