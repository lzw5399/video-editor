<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer UI -> preload/main/native API | User-selected file paths request a Rust-owned import mutation. |
| Rust report -> renderer panel | Adapter diagnostics become product-facing copy. |
| UI workflow -> preview/export | User-visible success is proven through product preview/export paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-33 | Tampering | Renderer import request | mitigate | Renderer sends only narrow path/session/revision request; Rust validates and mutates. |
| T-17-34 | Information Disclosure | Report panel | mitigate | Display bounded product copy; do not show raw formula JSON, secrets, signed URLs, or account identifiers. |
| T-17-35 | Repudiation | UI import success | mitigate | Product E2E requires report, timeline, preview/export, clean saved project, and no-fallback assertions. |
| T-17-36 | Tampering | Stale session UI mutation | mitigate | UI uses current project-session revision; Rust stale-revision gate remains authoritative. |
</threat_model>
