## Product And Language Contract

- First screen MUST be the actual editor workspace, not a landing page, dashboard, marketing hero, or project picker.
- Visible language MUST be Simplified Chinese by default for panel titles, controls, empty states, error text, accessibility labels used by tests, and Playwright-visible copy.
- UI vocabulary MUST use Jianying-style concepts: `草稿`, `素材`, `轨道`, `片段`, `源时间`, `目标时间`, `主轨磁吸`, `关键帧`, `贴纸`, `文字`, `特效`, `转场`, `滤镜`, `调节`.
- User-facing copy MUST NOT use `Asset`, `Clip`, `media bin`, `workbench`, or other English editor jargon.
- The app MUST read as a compact desktop video editor, not a generic admin dashboard. Do not use card-heavy page sections, decorative orbs, gradient hero sections, marketing copy, or oversized empty illustrations.

## Workspace Layout

Minimum verified desktop viewport: `1280x800`. Constrained minimum verified viewport: `1120x720`.

| Region | Required Size | Content Contract |
|--------|---------------|------------------|
| Top feature categories | `52px` height, full width | Product name plus category buttons: `媒体`, `音频`, `文字`, `贴纸`, `特效`, `转场`, `滤镜`, `调节`. Active category has fixed dimensions and accent indicator. Disabled/deferred categories remain visible with muted styling and Chinese empty states. |
| Left material/function panel | `300px` width at 1280px, `260px` minimum | Shows the active category. `媒体` is primary MVP. Material rows show name, type, duration, resolution/audio detail, and status. |
| Center preview monitor | `minmax(420px, 1fr)` | Stable black monitor shell with draft/binding status and preview placeholder. Real deterministic preview frames remain Phase 5. |
| Right inspector | `300px` width at 1280px, `260px` minimum | Reflects current selection. No selection shows useful empty state. Segment selection exposes only Phase 3-supported fields. |
| Bottom timeline | `260px` height at 1280x800, `220px` minimum | Multi-track timeline with ruler, transport strip, track headers, stable rows, and segment blocks. |

Layout grid: `300px minmax(420px, 1fr) 300px` columns and `52px minmax(0, 1fr) 260px` rows at the primary viewport. Use `1px` dividers between major regions. No region may overlap, collapse, or scroll the entire page; scrolling stays inside panels or timeline lanes.

## Component Behavior Contract

| Component | Contract |
|-----------|----------|
| Feature categories | Fixed-height buttons, no layout shift on hover/active. Categories are `媒体`, `音频`, `文字`, `贴纸`, `特效`, `转场`, `滤镜`, `调节`. Deferred categories open a panel-level empty state rather than disappearing. |
| Material rows | Stable `72px` minimum height. Show Chinese type labels: `视频`, `图片`, `音频`, `文字`, `贴纸`. Status labels: `可用`, `素材丢失`, `解析失败`. Rows do not expand on hover. |
| Preview monitor | Shows a `16:9` monitor frame centered in its region. Phase 04 placeholder copy: `预览将在下一阶段接入`; status line may show binding/draft state in Chinese. |
| Inspector fields | Fixed label column, numeric inputs for microsecond-backed values, segmented control for text alignment, swatches for color values, toggle for track mute. Fields call commands on explicit commit or controlled debounced commit, not on every uncontrolled local mutation. |
| Timeline ruler | Fixed `28px` height. Time labels use integer microseconds formatted as `00:00:00.000` for display only. Rendering/edit semantics remain Rust-owned. |
| Track rows | Fixed row heights: video/text/sticker/filter `56px`, audio `48px`. Track headers fixed `160px` width with mute/lock affordances. Hover/selection must not resize rows. |
| Segment blocks | Rounded radius `4px`, fixed minimum visible width `8px`, selected outline uses accent. Text inside segment blocks truncates; it must never overflow neighboring segments. |
| Transport controls | Fixed `36px` controls; play/pause can be disabled or placeholder in Phase 04 because real preview playback is Phase 5. |
| Errors | Inline command errors appear in the region where the failed action occurred plus optional global status strip. The UI keeps the previous draft state after Rust rejects a command. |

## Command Boundary And State Contract

- Renderer MUST call `window.videoEditorCore.executeCommand` with generated `CommandEnvelope` payloads for material, timeline, text, and audio actions.
- Renderer MUST consume generated `CommandResultEnvelope<T>` and `TimelineCommandResponse` as the only accepted state update path after timeline commands.
- Renderer may hold current `Draft`, `CommandState`, and `TimelineSelection` for display and for the next command payload, but MUST NOT directly mutate `draft.tracks`, `track.segments`, `segment.targetTimerange`, `segment.sourceTimerange`, undo/redo stacks, snapping state, or main-track magnet behavior.
- Renderer MUST NOT construct FFmpeg/ffprobe commands, render graphs, preview cache paths, waveform paths, or export scripts.
- Invalid command results MUST show Chinese error copy and preserve the last accepted draft state. The UI must not locally repair or retry rejected edits in a way that hides the Rust error.
- MVP drag editing may be represented by deterministic buttons/inputs first, but every accepted add/select/move/split/trim/delete/text/audio/volume/mute action must be proven through Rust command responses.

## Verification Expectations

- Playwright Electron MUST verify the app opens directly to a Chinese editor workspace with visible regions named `顶部功能区`, `素材面板`, `预览窗口`, `属性检查器`, and `时间线`.
- Visual/layout checks MUST run at `1280x800` and `1120x720`, verifying the top categories, left panel, preview, inspector, and timeline are visible, non-overlapping, and not clipped.
- Tests MUST cover material rows in Chinese, including `可用`, `素材丢失`, and `解析失败` states when fixtures permit.
- Tests MUST prove at least one timeline edit updates UI state through `window.videoEditorCore.executeCommand` and `TimelineCommandResponse`.
- Source guards MUST reject direct renderer mutation patterns for `Draft.tracks`, `Track.segments`, generated contract edits, direct Electron/Node imports in renderer, FFmpeg/ffprobe construction in renderer, and English-only key labels for Phase 04 workspace regions.
- Existing gates `just build` and `just test` remain required; Phase 04 must add workspace-specific scripts for source guards and Electron/visual checks.
