## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Timeline UI event -> generated command | User timeline actions cross from renderer controls into Rust command validation. |
| TimelineCommandResponse -> renderer state | Rust response is the only accepted source for draft, command state, and selection updates. |
| Renderer display math -> semantic timeranges | UI position/width calculations are display-only and must not become persisted semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Tampering | Timeline controls | mitigate | Build commands from generated payloads and replace accepted state only from `TimelineCommandResponse`. |
| T-04-12 | Tampering | Snapping/main-track magnet | mitigate | Renderer does not compute snapping or magnet candidates; it displays Rust command events. |
| T-04-13 | Repudiation | Undo/redo | mitigate | Renderer calls `undoTimelineEdit`/`redoTimelineEdit`; it does not store or apply inverse operations locally. |
| T-04-14 | DoS | Timeline layout | mitigate | Fixed row/header/ruler/segment/transport dimensions and non-resizing hover/selection states. |
| T-04-15 | Tampering | Invalid edit handling | mitigate | Rejected commands keep previous accepted state and display Chinese Rust error copy. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install is planned; use existing dependencies only. |
</threat_model>
