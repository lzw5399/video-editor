## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer -> Electron main | Untrusted renderer IPC requests scheduler status/telemetry only. |
| Electron main -> native binding | Main forwards typed requests after sender validation; dev config overrides are diagnostics-gated outside default renderer APIs. |
| Native binding -> scheduler | Binding applies typed dev config and returns aggregate status/telemetry. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-05A | Tampering | scheduler config mutation | mitigate | Rust validation and hard caps; config override is native/main/dev diagnostics only and not exposed through default preload/renderer APIs. |
| T-16-05B | Information Disclosure | scheduler telemetry UI | mitigate | Product-safe aggregate fields by default; raw internals only behind developer diagnostics. |
| T-16-05C | Spoofing | scheduler/product readiness | mitigate | Scheduler telemetry never substitutes for `renderGraphGpuComposited` product preview evidence. |
| T-16-05D | Denial of Service | config abuse | mitigate | Reject zero/huge/unbalanced resource budgets and keep hard Rust-side caps for desktop development overrides. |
</threat_model>
