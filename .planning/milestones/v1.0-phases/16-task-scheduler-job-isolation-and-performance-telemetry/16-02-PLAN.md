<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Electron/bindings -> preview scheduler | UI requests preview/audio work but cannot own cadence, retry, or resource policy. |
| Scheduler -> compositor/audio output | Completion crosses into visible output and must pass freshness/cancel gates. |
| Background load -> interactive lanes | Export/artifact/probe saturation must not deny realtime preview/audio service. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02A | Denial of Service | preview/audio admission | mitigate | Dedicated realtime/interactive scheduler lanes, explicit resource budgets, and starvation tests. |
| T-16-02B | Tampering | stale preview/audio completion | mitigate | Require `PlaybackGeneration` and target microseconds; reject stale completions before present/refill and increment telemetry. |
| T-16-02C | Spoofing | product preview success | mitigate | Preserve `renderGraphGpuComposited` evidence and reject fallback/mock/artifact/DOM-only success in tests. |
| T-16-02D | Information Disclosure | preview/audio diagnostics | mitigate | Raw queue internals stay out of product UI; only product-safe exception copy and aggregate telemetry move forward. |
</threat_model>
