<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binding artifact/probe API -> scheduler | Product commands request heavy artifact/probe/IO work; scheduler owns admission and cancellation and returns pending probe status without blocking command responsiveness. |
| Scheduler -> media/artifact/project runtimes | Scheduler invokes existing runtime executors and store operations under resource budgets. |
| Generated/probed result -> draft/artifact-visible state | Completion can mutate project-visible state only after identity, revision, freshness, and cancellation checks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-04A | Denial of Service | artifact/probe/IO queues | mitigate | Bounded background/IO/probe resource classes, classified rejection, queued/pending probe responses, and starvation/responsiveness tests. |
| T-16-04B | Tampering | stale artifact/probe commits | mitigate | Expected revision and generation checks before blob, row, material, or project-state mutation. |
| T-16-04C | Spoofing | artifact fallback evidence | mitigate | Guards and tests prevent thumbnails/waveforms/probe diagnostics from satisfying product preview success. |
| T-16-04D | Information Disclosure | raw probe/process output | mitigate | Keep bounded diagnostic summaries; product responses expose classified messages only. |
</threat_model>
