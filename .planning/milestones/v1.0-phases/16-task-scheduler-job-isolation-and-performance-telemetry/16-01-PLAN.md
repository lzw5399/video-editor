<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Domain crates -> `task_runtime` | Preview/audio/export/artifact/probe crates begin depending on a Rust-owned scheduler boundary. |
| Freshness contracts -> visible state | Later completion handlers will trust these types before mutating preview/audio/artifact/material-visible state. |
| Desktop integration -> portable runtime | Desktop crates may depend on portable contracts but cannot encode renderer or desktop process policy into freshness types. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01A | Tampering | `PlaybackGeneration` ownership | mitigate | Keep exactly one generation type and verify with a source count gate. |
| T-16-01B | Denial of Service | workspace dependency graph | mitigate | Add only local dependency edges needed for scheduler integration and run focused cargo checks. |
| T-16-01C | Information Disclosure | freshness serialization | mitigate | Exclude filesystem paths, FFmpeg paths, worker names, and renderer-controlled policy fields from freshness contracts. |
| T-16-01SC | Tampering | npm/pip/cargo installs | accept | No new external package installs are planned; use existing Rust/std and workspace crates. |
</threat_model>
