## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> project_store | Local `.veproj/project.json` content and paths cross into Rust persistence. |
| project_store -> draft_model | Loaded JSON is migrated and validated by semantic Rust model. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | `.veproj/project.json` loader | mitigate | Strict serde/schema/migration validation; malformed JSON or unknown schema versions return structured errors and do not produce partial drafts. |
| T-02-06 | Tampering | Save/open round trip | mitigate | Semantic equality tests per D-05; never compare only raw bytes or silently normalize invalid semantics. |
| T-02-07 | Tampering/Information Disclosure | Material URI path resolution | mitigate | Centralize relative/external URI classification in `project_store`; reject traversal/ambiguous paths with structured errors. |
| T-02-08 | Tampering | Project-store responsibility boundary | mitigate | Grep and tests prove `project_store` does not call ffprobe, `media_runtime`, or material import registry orchestration. |
| T-02-09 | Tampering | Missing material persistence | mitigate | Open/save preserves entries when media files are absent; missing-file state is recoverable and not a reason to delete material records per D-15. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package-manager install is planned. If one becomes necessary, stop for a blocking human package legitimacy checkpoint. |
</threat_model>
