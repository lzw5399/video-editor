<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer UI -> generated command envelope | User audio interactions become generated commands to Rust. |
| Rust response -> renderer display model | Audio status, devices, waveform peaks, and parity warnings become safe display state. |
| Waveform peaks -> timeline visuals | Derived display data enters segment rendering without affecting canonical audio semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-05-01 | Tampering | `commandHelpers.ts`, `App.tsx` | mitigate | Helpers only build generated command envelopes; App applies accepted Rust-shaped responses and preserves prior state on rejection. |
| T-15-05-02 | Information Disclosure | production UI copy | mitigate | UI tests and final guards reject internal terms, native backend names, session IDs, artifact paths, SQLite, fingerprints, dirty ranges, and FFmpeg filter text. |
| T-15-05-03 | Denial of Service | waveform display | mitigate | Renderer consumes bounded peak arrays and CSS-fixed waveform dimensions; no renderer path or DB reads. |
| T-15-05-04 | Spoofing | mock audio commands | accept | Test-only mocks are gated by `VIDEO_EDITOR_TEST_MOCK_AUDIO_COMMANDS=1`; production bridge still calls Rust binding. |
</threat_model>
