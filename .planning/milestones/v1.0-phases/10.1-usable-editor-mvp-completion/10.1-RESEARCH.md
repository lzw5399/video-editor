# Phase 10.1: Usable Editor MVP Completion - Research

**Date:** 2026-06-18
**Status:** Initial codebase gap audit

## Current Code Evidence

### Already Available

- Rust/generated command surface includes `importMaterial`, `addSegment`, `selectTimelineSegments`, `moveSegment`, `splitSegment`, `trimSegment`, `deleteSegment`, `undoTimelineEdit`, `redoTimelineEdit`, `addTextSegment`, `editTextSegment`, `importSubtitleSrt`, `addAudioSegment`, `setSegmentVolume`, `setTrackMute`, `updateSegmentVisual`, `requestPreviewFrame`, and export commands.
- `apps/desktop-electron/src/renderer/App.tsx` already centralizes command execution through `window.videoEditorCore.executeCommand`.
- `FeaturePanel.tsx` already has material, text/SRT, and audio panels, but media import still exposes manual `草稿包路径` and `素材路径` inputs.
- `PreviewMonitor.tsx` already stores `preview.frameArtifactPath`, but the visible canvas currently shows status text and a code path instead of the PNG image.
- `Timeline.tsx` already renders a vertical `.playhead`, selection state, track headers, mute buttons, and toolbar actions for add/move/split/trim/delete/undo/redo. Playhead changes are mostly input/button based; ruler click/drag is missing.
- `Inspector.tsx` already contains draft parameters, visual transform controls, text controls, keyframe controls, and audio volume/mute callbacks.
- Phase 09 Rust command support already parses SRT in `crates/draft_commands/src/text.rs` and routes it through bindings.

### Main Gaps

- No Electron main/preload file picker API exists. `preload/index.ts` exposes only `videoEditorCore`; the user must type paths.
- `videoEditorCore` smoke test asserts exactly `ping`, `version`, and `executeCommand`, so platform dialogs should use a separate bridge such as `window.videoEditorPlatform.openMaterialFiles()`.
- Preview frame artifacts are not displayed as image pixels. Tests currently assert that the path text appears in `预览画面`, which must change.
- Real preview image display needs an Electron-safe artifact URL. A direct `file://` path in a sandboxed renderer can be brittle; main/preload should expose either a safe path-to-URL helper or a protocol/IPC read path. For P0 desktop, converting returned artifact paths to `file://` in preload is acceptable if sender origin is still enforced and only local artifact paths returned by commands are consumed.
- Playhead changes do not automatically request preview frames. This phase should make seek actions call `onRequestPreviewFrame` or a dedicated `onSeekPreview`.
- Timeline ruler and playhead are not clickable/draggable yet.
- Existing derived preview invalidation after visual/text/keyframe edits clears stale state but does not automatically request a fresh frame.
- Copy/duplicate has no obvious generated Rust command. Do not implement renderer-owned copy; document as deferred unless a Rust command exists by execution time.
- Playwright coverage currently uses manual path inputs and mocked preview responses. New tests should add file chooser IPC coverage and image/canvas display checks.

## Implementation Approach

1. Add `videoEditorPlatform` bridge:
   - main: `dialog:openMaterialFiles` handler with allowed sender check;
   - preload: expose `openMaterialFiles()` and `pathToFileUrl(path)` if needed;
   - renderer: no Electron/Node imports.
2. Update import UX:
   - `导入素材` opens file picker first;
   - for each selected path, call existing `importMaterial`;
   - keep typed paths as collapsed fallback or test utility if needed.
3. Update preview display:
   - store both artifact path and display URL in `PreviewDisplayState`;
   - render `<img alt="当前预览帧">` inside `.preview-canvas`;
   - preserve black canvas fallback and path diagnostics below.
4. Update playhead interactions:
   - add seek helper in `App.tsx` that updates playhead and requests preview;
   - timeline ruler click and playhead drag compute integer microseconds from pointer position and timeline duration;
   - previous/next frame uses draft frame rate to step exact frame duration.
5. Add tests and guards:
   - Playwright can mock Electron `dialog.showOpenDialog` in main process or use a test-only env path provider; the app code must still call main/preload dialog.
   - Source guards should block renderer Electron/Node imports, direct draft mutation, FFmpeg/render graph construction, direct subtitle cue construction, and preview/waveform/export derived artifact persistence.

## Risk Notes

- Electron native dialogs are awkward in automated tests. Prefer a main-process test hook around `dialog.showOpenDialog` rather than a browser `<input type=file>`, because the product architecture requires Electron main/preload file chooser ownership.
- Preview image rendering may need CSP/path handling if packaged file loading blocks local artifacts. If `file://` proves brittle, move to a custom `video-editor-preview://` protocol or IPC bytes/data URL in a later slice.
- Importing real materials depends on FFmpeg/ffprobe availability. Playwright real workflow gates should skip or fail clearly when runtime diagnostics report preview/import unavailable.
