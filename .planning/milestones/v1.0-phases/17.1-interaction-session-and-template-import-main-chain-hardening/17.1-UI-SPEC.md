## Visual Hierarchy

Primary focal point is the preview canvas and selected overlay because it shows the result of live Rust provisional interaction. Secondary focal point is timeline/playhead state because it explains timing, track legality, and commit scope. Tertiary focal point is the inspector and template report because they provide precise parameter edits and diagnostic navigation without competing with the canvas.

## Interaction Ownership

Rust owns: draft/timeline/selection validity, base revision, interaction lifecycle, provisional view model, provisional deltas, preview generation IDs, stale rejection, cancel, final commit, revision increment, save/autosave, and undo item creation.

React owns only: pointer capture, hover/focus styling, same-animation-frame local affordance before the latest Rust provisional result arrives, input text composition UI, scroll/zoom viewport, and disabled/pressed visual states. React must discard local affordances on Rust provisional rejection, cancel, commit, stale generation, selection change, or project revision change.

Prohibited: high-frequency draft mutation through repeated canonical `executeProjectIntent`/timeline intent loops; TypeScript intent-name switches for preview invalidation; DOM-only ghost previews as success evidence; renderer-owned snapping, track legality, crop bounds, keyframe interpolation, undo/redo, project revision, save, render graph, or preview cache semantics.

## Shared Interaction States

Every draft-mutating high-frequency interaction uses:

`Idle -> Begun -> Streaming -> CommitPending -> Committed`

Allowed exits: `Begun/Streaming -> Canceled`; `Begun/Streaming/CommitPending -> StaleRejected`; validation failure returns to `Idle` without draft mutation.

Required UI state mapping:

| State | Visual Contract | Semantic Contract |
|-------|-----------------|-------------------|
| Begun | pointer target is active; selected outline remains stable | Rust returns `interactionId`, `baseRevision`, generation, `acceptedSequence: 0` |
| Streaming | local affordance may lead by one frame; accepted Rust provisional result is drawn as soon as received | updates are rAF-throttled, monotonic, one in flight, coalesced by Rust |
| CommitPending | control shows pending affordance without layout shift; no duplicate commit controls | no additional samples accepted except final sequence reconciliation |
| Committed | accepted view model replaces affordance; one undo item visible through normal undo state | one canonical mutation, one revision increment, one save/autosave decision, one delta-driven invalidation |
| Canceled | local affordance removed; selected state restored | no revision, no save, no undo, no project file change |
| StaleRejected | local affordance removed; show stale copy once in nearby status/error location | no revision, no save, no undo, no project file change |

Navigation-only interactions use the same coalescing/backpressure and generation rejection rules but do not create draft mutations, saves, revisions, or undo items.

## Surface Contracts

| Surface | Required Behavior | Local UI Affordance | Rust-Owned Result |
|---------|-------------------|---------------------|-------------------|
| Preview canvas move | Drag selected video/image/text overlay in canvas coordinates; supports text overlay hit test then drag | selected outline follows pointer for same frame | validated position, selection handle, provisional view model, delta, preview generation |
| Preview canvas rotate | Rotate handle streams degrees while dragging | handle remains visible; angle ghost may render for same frame | validated degrees; no UI-owned interpolation |
| Preview canvas scale/resize | Expose scale/resize handles only when implemented through interaction sessions | handles visible on selected outline with tooltips | validated `scaleX/scaleY`; preview updates from provisional state |
| Preview canvas crop/text-box resize | Hide or gate until Rust provisional crop/text layout and preview/export support exist | disabled tooltip only if shown | unsupported controls cannot look functional |
| Timeline move | Drag segment within track; Rust owns snapping/magnet and collision legality | segment may translate under pointer; target track highlight is only a hint | accepted start time in integer microseconds, snap diagnostics, delta |
| Timeline cross-track move | Drag selected segment across compatible tracks | candidate track highlight; locked/incompatible tracks show rejected cursor/state | Rust validates track kind, lock, visibility/mute constraints, stacking |
| Timeline trim | Left/right trim streams edge time; 16px effective handle target required | width/edge ghost may render until provisional response | Rust validates bounds/source range; one commit on release |
| Playhead scrub | Ruler click-drag, playhead drag, wheel/trackpad scrub, frame-step repeat, report navigation seeks | playhead line moves immediately but marks pending when preview generation lags | coalesced seek with target integer microseconds, audio sync, stale-frame rejection |
| Inspector visual | Position, scale, rotation, opacity, crop sliders/numbers stream provisional edits | range thumb and number update immediately | Rust validates ranges and returns provisional view model/delta; blur/Enter may commit discrete final value |
| Inspector text | Content, font size, color, stroke, shadow, background, line height, letter spacing, box/layout controls avoid per-sample undo pollution | composition text can stay local while composing | Rust owns validation, fontRef, text layout, keyframed values; one commit per completed interaction |
| Inspector audio | Gain, pan, fade sliders/numbers stream provisional audio edits | sliders update immediately; waveform remains derived/status-only | Rust/audio graph owns gain, pan, fades, preview sync, delta |
| Keyframe lane/markers | Marker drag changes segment-relative time; nearest-keyframe navigation/delete is supported | diamond marker has visible focus/drag state | Rust validates integer microsecond time, property support, interpolation/easing |
| Template report navigation | Row click focuses material/track/segment/text/keyframe/filter/resource when canonical target exists and seeks when time exists | selected row highlight and keyboard roving focus | coalesced navigation/focus/seek; report-only state if no editable target |

## Product Units

Default UI must display:

| Domain | UI Unit | Persisted/Semantic Unit |
|--------|---------|-------------------------|
| Timecode | `HH:MM:SS:FF` or `MM:SS.ff` style product time; no raw microsecond labels in default controls | integer microseconds / frame index |
| Fade durations | seconds, step `0.1s`, range `0-60s` | integer microseconds |
| Scale | percent, range `0.1%-300%`, step `1%` | millis or typed Rust scale value |
| Opacity | percent, range `0%-100%`, step `1%` | millis or typed Rust opacity value |
| Crop | percent per edge, range `0%-99.9%`; opposite sides must total `<100%` | millis or typed Rust crop value |
| Rotation | degrees, range `-360deg` to `360deg`, step `1deg` | typed degrees |
| Position | canvas-centered X/Y product coordinates; label direction `X 向右，Y 向上` | typed Rust coordinate value |
| Text line height | percent, default `120%`, range `50%-300%` | millis |
| Letter spacing | px or percent only if preview/export supports that displayed unit | typed Rust text spacing value |
| Canvas fps | integer preset or rational custom, displayed as `24 fps`, `30000/1001 fps`, etc. | rational frame rate |

Developer diagnostics may show raw microseconds, generation, backend, fallback, graph/cache, and runtime labels only when developer mode is enabled.

## Unsupported-Control Gating

Default visible controls must be one of:

1. Fully functional through production preview/export semantics.
2. Disabled with unavailable copy and no active hover/pressed styling.
3. Hidden from default mode.

Sticker/effect/transition/filter/adjustment showcase surfaces must not present cards as insertable/applicable unless Rust command, preview, export, and report behavior exist. If visible as deferred catalog placeholders, they must be inert, disabled, and labeled through product copy such as `暂不可用`, not styled as active resources.

`image` background fill, crop handles, transitions/fades, effects, masks, blend modes, and proprietary template effects must be gated unless Phase 17.1 implementation proves production preview/export support. Unsupported report rows navigate only to the report row and show `此项没有可编辑的草稿目标`.

## Accessibility

- Every pointer interaction has an accessible name matching the visible product concept: `移动片段`, `旋转文字`, `左侧裁剪`, `右侧裁剪`, `拖动播放头`, `移动关键帧`, `跳转到适配条目`.
- Keyboard parity required for selection, report row navigation, playhead frame-step, timeline delete/split, keyframe add/remove, and inspector numeric commit.
- Focus rings use cyan border/outline and must not shift layout.
- Disabled controls remain focus-skipped unless there is a needed explanatory tooltip; unavailable state must be conveyed by `aria-disabled` or `disabled` plus `title`.
- Status changes for stale/rejected/committed/canceled interactions use one nearby `aria-live="polite"` region; do not spam live regions during every pointer sample.
- Hit targets: trim/playhead/keyframe marker effective target 16px minimum, icon buttons 24px or 32px, inspector controls 32px height.

## Verification Contract

Required screenshot viewports:

| Viewport | Required Evidence |
|----------|-------------------|
| `1280x800` | Five-zone workspace visible, template report list navigable, inspector units normalized, preview handles visible, no debug-only copy |
| `1120x720` | No text overlap, timeline controls stable, inspector rows fit, report rows remain readable, timeline trim/keyframe hit areas usable |

Required no-fallback evidence:

- Preview canvas interaction tests must prove compositor pixels change with edit state, not just DOM outline movement.
- Timeline move/trim/cross-track tests must fail if only local ghost state changes before commit.
- Inspector slider tests must fail if each sample creates a canonical command, revision, save, or undo entry.
- Playhead scrub tests must fail when stale preview generations can present after a newer seek/cancel.
- Template import tests must prove `CommandDelta` drives invalidation; hardcoded TypeScript intent switches are not accepted.
- Product E2E must fail closed when the production realtime preview path is unavailable; mock/native fallback/artifact/first-frame evidence is not success.

Required source guards:

- No high-frequency draft-mutating path may call repeated canonical `executeProjectIntent` loops.
- No default-mode UI may expose FFmpeg/ffprobe paths, graph/cache internals, provider raw data, backend diagnostics, or fallback success copy.
- Unsupported visible controls must be disabled/hidden/gated and covered by screenshot evidence.
