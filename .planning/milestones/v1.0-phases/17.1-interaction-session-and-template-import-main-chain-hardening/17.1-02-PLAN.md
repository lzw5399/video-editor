## Decision Coverage

- D-03: Make Rust own validation, provisional state, final commit, and preview invalidation facts.
- D-04: Define commit as the only canonical mutation/revision/save/undo point.
- D-05: Require live updates to avoid save, revision, undo, and redo mutation.
- D-06: Test cancel and stale rejection as no-op paths for canonical project state.
- D-07: Enforce `baseRevision` equality on commit.
- D-08: Give high-frequency sample streams a first-class interaction-session API.
- D-09: Allow draft-mutating and navigation-only interactions to share coalescing semantics without all becoming draft saves.
- D-10: Keep discrete commands outside the high-frequency session path.
- D-11: Implement the `ProjectInteractionSession` state machine.
- D-12: Expose explicit begin/update/commit/cancel IPC.
- D-13: Require monotonic sequence handling and accepted/coalesced sequence responses.
- D-14: Require requestAnimationFrame throttling and one in-flight update on the renderer side while Rust remains authoritative.
- D-15: Carry provisional generation data for stale preview rejection.
- D-16: Align updates with scheduler/backpressure coalescing.
