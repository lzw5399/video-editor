<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| command delta -> preview cache | Dirty facts decide cache reuse and invalidation. |
| preview/export service -> binding payloads | Rust-owned dirty facts are later transported by bindings without renderer recomputation. |
| export preparation -> FFmpeg compiler | Export prep receives dirty facts, while FFmpeg compiler still only compiles jobs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-13 | Tampering | `PreviewCacheKey` v2 | mitigate | Derive key fields from Rust graph/fingerprint helpers and reject renderer-computed key logic with source guards. |
| T-13-14 | Tampering | invalidation predicate | mitigate | Test full-draft, range, material, graph node, domain, and runtime/profile invalidation paths. |
| T-13-15 | DoS | export prep invalidation | mitigate | Keep Phase 13 as classification facts only; scheduling and background work remain out of scope per D-05. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installation is required. |
</threat_model>
