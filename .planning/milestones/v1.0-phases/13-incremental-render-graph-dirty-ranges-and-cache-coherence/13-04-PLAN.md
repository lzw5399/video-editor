<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| engine normalized state -> render graph | Graph IDs derive from semantic entities and roles. |
| graph snapshot -> cache/export consumers | Fingerprints drive reuse decisions and must not be confused with identity. |
| runtime capability fingerprint -> graph fingerprints | Runtime changes invalidate relevant derived outputs without changing semantic node IDs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-10 | Tampering | `RenderGraphNodeId` | mitigate | Tests assert stable keys never include content hashes and remain stable across content edits. |
| T-13-11 | Tampering | `RenderGraphNodeFingerprint` | mitigate | Keep fingerprints separate from IDs and include schema/generator/runtime fields. |
| T-13-12 | DoS | graph diff | mitigate | Use map/set comparisons by stable key and test large localized diffs in Plan 13-06. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installation is required. |
</threat_model>
