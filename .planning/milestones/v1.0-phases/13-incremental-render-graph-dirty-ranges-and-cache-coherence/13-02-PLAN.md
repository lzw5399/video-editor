## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Electron binding -> `draft_commands` | Command payloads are untrusted until Rust validation accepts them. |
| `draft_commands` -> preview/render consumers | Delta facts drive derived invalidation and must not under-report dirty ranges. |
| generated TypeScript -> renderer | Renderer receives delta data after Plan 13-02B but must not compute it per D-06. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Tampering | `CommandDelta.changed_ranges` | mitigate | Use checked `Microseconds` arithmetic and include old plus current ranges for timing edits. |
| T-13-05 | DoS | range merge helpers | mitigate | Sort and merge deterministically with checked ends; reject or full-invalidate on overflow. |
| T-13-06 | Repudiation | response contract migration | mitigate | Tests assert every accepted simple command response contains a delta with command and reason. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installation is required. |
</threat_model>
