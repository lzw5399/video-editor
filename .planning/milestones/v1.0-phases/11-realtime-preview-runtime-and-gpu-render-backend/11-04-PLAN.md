<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main process -> native binding | Main passes native window handles and surface bounds to Rust through Node-API. |
| native binding -> runtime surface | Rust validates handles, bounds, lifecycle, and generation before presentation. |
| runtime surface -> GPU backend | Surface descriptors cross into `wgpu` target setup without exposing GPU internals outside Rust. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-10 | Denial of service | native surface bounds | mitigate | Validate nonzero bounds and scale factor; detach/close surfaces on window close and surface errors. |
| T-11-11 | Information disclosure | native handles and GPU state | mitigate | Do not expose `HWND`, `NSView`, GPU device, surface, or command encoder handles to TypeScript. |
| T-11-12 | Tampering | binding session registry | mitigate | Use opaque session IDs, reject malformed IDs, and keep session lifecycle in Rust-owned bindings. |
| T-11-SC | Tampering | platform dependencies | mitigate | Use packages approved in `11-RESEARCH.md`; no platform dependency may be added without a package legitimacy row. |
</threat_model>
