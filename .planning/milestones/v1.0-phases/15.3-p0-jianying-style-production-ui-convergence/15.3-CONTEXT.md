# Phase 15.3: P0 Jianying-Style Production UI Convergence - Context

**Gathered:** 2026-06-20
**Status:** Ready for planning
**Source:** User-directed context from UI alignment, real playback, no-fallback, and product E2E feedback.

<domain>
## Phase Boundary

Phase 15.3 converts the current desktop editor from an engineering-heavy workspace into a production Jianying-style editor shell. The feature set stays smaller than Jianying Pro, but the default interaction model must match the reference hierarchy: project entry first, top feature/export bar, left resource panel, center preview/player, right contextual inspector, and bottom timeline.

This phase does not add new editing semantics to Rust. It makes existing supported semantics usable through a production desktop UI and locks them with user-level Playwright/Electron regression tests.
</domain>

<decisions>
## Implementation Decisions

### D-01 Production UI, Not Debug Console
- Default UI must not expose FFmpeg, ffprobe, runtime/backend telemetry, artifact/cache paths, graph/cache internals, raw diagnostics, manual path fields, or developer-only status. These can exist only behind explicit developer diagnostics.

### D-02 Jianying-Style Interaction Shape
- Use Jianying Pro as the layout and interaction reference, with functional reduction only. The app may have fewer features, but the visible structure and usage pattern must stay familiar: top primary feature tabs, left feature content, center preview, right inspector, bottom timeline.

### D-03 Project Entry Before Editing
- Normal users must create a new project or open an existing `.veproj` before importing materials. The editor workspace should not be the default first surface for blank product launch.

### D-04 Export Is A Top-Right Modal
- Export moves out of the preview monitor. The top-right product bar owns the Export action and opens a modal with path, resolution, frame rate, bitrate, audio options, progress, cancel, and open-location controls.

### D-05 Real Playback Acceptance
- Playback is not accepted when only the playhead/timecode advances. Normal user tests must assert real `renderGraphGpuComposited` evidence and visible preview-region pixel motion. `requestPreviewFrame`, preview artifacts, mock displays, first-frame stills, CPU probes, native single-video layers, or fallback paths cannot satisfy playback success.

### D-06 Product E2E Is The Standard
- User-visible behavior must be tested by normal Playwright/Electron workflows: create/open project, import repo media fixtures, add or drag materials to timeline, edit through visible controls, preview/play, save/reopen, and export. Unit tests alone are not sufficient for user-facing completion.

### D-07 No Fallback And No Legacy Compatibility By Default
- This greenfield editor should replace obsolete partial implementations instead of preserving old product paths. Unsupported product controls must be hidden, gated, or fail closed with product-safe copy; they must not route to fallback/mock/legacy behavior.

### D-08 Screenshot References Are Provisional
- `docs/ui-reference/jianying-pro/screenshots/` is useful for layout and interaction hierarchy, but the current 19-file batch is provisional and title/state mismatches are suspected. Phase 15.3 must not treat it as a pixel-perfect golden set until a manifest-backed recapture exists.

### D-09 Icon Sourcing
- New production icons must first be selected from `/Users/zhiwen/code/video-editor/icons`, then copied into the application assets before use. Generate new SVGs only if no suitable source icon exists.

### D-10 Digital Human Out Of Scope
- 数字人 is not relevant for this phase. Keep the category out of default product focus or explicitly mark it as deferred if it remains visible.

### D-11 Renderer Boundary
- UI controls may collect user intent and present command responses, but renderer code must not construct FFmpeg commands, render graphs, cache keys, GPU commands, timeline semantics, or draft mutation logic.
</decisions>

<specifics>
## Specific Ideas

- Add a production entry shell with "新建项目" and "打开项目" before material import is available.
- Keep developer diagnostics available for tests and debugging through `VIDEO_EDITOR_TEST_SHOW_DEVELOPER_DIAGNOSTICS=1`, but default product E2E must run with it disabled.
- Replace text glyph toolbar icons with copied SVG assets where practical, especially playback, undo/redo, split, delete, zoom, snapping, track state, and export.
- Export modal must cover collapsed and expanded/advanced states, plus dropdown/open-section behavior in tests.
- Draft/canvas parameter editing should open a focused modal when editing broader draft settings, matching the user-observed Jianying pattern.
- Timeline default toolbar must emphasize direct editing interactions. Numeric move/split/trim controls are developer/test conveniences and should not be visible by default.
</specifics>

<deferred>
## Deferred Ideas

- Pixel-perfect Jianying clone and proprietary asset/icon copying are out of scope.
- 数字人 is out of scope.
- New retiming/effects/transitions semantics remain Phase 18 scope.
- Full screenshot recapture from Jianying Pro is required before treating references as strict golden comparisons; Phase 15.3 can create the manifest and regression harness.
</deferred>
