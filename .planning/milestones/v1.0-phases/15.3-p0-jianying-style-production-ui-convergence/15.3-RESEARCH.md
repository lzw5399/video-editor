# Phase 15.3 Research

## Question

What needs to be known to plan the Jianying-style production UI convergence without weakening the real GPU playback and no-fallback guarantees from Phase 15.2?

## Findings

### Current UI Shape

- `WorkspaceShell.tsx` already owns the five-zone layout and top category row.
- `FeaturePanel.tsx` already separates media/text/audio/deferred categories, but the media panel still shows resource task/maintenance information by default.
- `PreviewMonitor.tsx` still owns export controls and exposes a permanent export panel inside the preview area.
- `PreviewMonitor.tsx` hides request-preview-frame and request-preview-segment behind developer diagnostics, but default preview still shows numeric `预览时间` and product realtime status text that needs stronger product wording.
- `Timeline.tsx` still exposes numeric move/split/trim/playhead inputs in the default toolbar; these are useful for tests but not Jianying-style product controls.
- `Inspector.tsx` is already contextual but needs draft/canvas modal behavior and default-mode cleanup.
- `App.tsx` has open/save project command wiring but no project-entry shell before import.

### Reference State

- `docs/ui-reference/jianying-pro/screenshots/` currently contains 19 provisional screenshots.
- The current reference docs explicitly warn that screenshot titles may not match UI state and that missing modal/dropdown states still need recapture.
- Therefore Phase 15.3 should use these screenshots for structure and density, not pixel-level golden comparison.

### Existing Product E2E Grounding

- `product-user-journey.spec.ts` already asserts real `renderGraphGpuComposited` playback evidence and visible preview-center pixel motion.
- `real-workflow.spec.ts` already covers dev/packaged real import, preview artifacts, export, save/reopen, and repo-owned fixtures, but it currently opts into developer diagnostics because export and preview artifact controls are developer-style surfaces.
- `workspace.spec.ts` already verifies five-zone layout at 1280x800 and 1120x720 and default-mode hiding of multiple diagnostics.

### Icons

- The repository has a source icon set under `/Users/zhiwen/code/video-editor/icons`.
- Useful existing candidates include:
  - `icons/MaterialReplacement/play.svg`, `pause.svg`;
  - `icons/ToolBar/undo.svg`, `redo.svg`, `del.svg`, `cutting.svg`, `btnZoomIn.svg`, `btnZoomOutN.svg`, `btnOpenAdsorb.svg`, `btnCloseAdsorb.svg`, `addTimeline.svg`;
  - `icons/Cursor/cutLeft.svg`, `cutRight.svg`, `movable_cursor.svg`;
  - `icons/segment/locked.svg`, `unlocked.svg`.
- Phase implementation should copy selected SVG files into `apps/desktop-electron/src/renderer/assets/icons/` and import from there.

### Production Risks

- A UI-only refactor can accidentally make tests pass while preserving debug/fallback paths in production. Source guards must scan default UI copy and normal product E2E.
- Moving export to a modal can break real workflow tests unless tests are updated to use the top-right modal instead of developer diagnostics.
- Hiding numeric timeline controls requires replacing test helpers with pointer/mouse interaction or product toolbar actions.
- Project entry changes will require product tests to create/open a project before import; old tests that start directly in workspace should become explicit fixtures/dev harnesses.

## Validation Architecture

Phase 15.3 verification must combine:

- Default-mode Playwright layout tests at 1280x800 and 1120x720.
- Normal user product journeys that start from project entry, import repo media fixtures, add/edit timeline content, play through the real compositor, save/reopen, and export from the top-right modal.
- Source guards that reject default UI/debug text and fallback product paths.
- Developer-diagnostics tests that prove hidden controls remain available only when diagnostics are explicitly enabled.
- Screenshot/reference tests that compare region presence, hierarchy, and no-overlap against the provisional Jianying reference set without treating the current reference filenames as final golden truth.

## Constraints For Plans

- Do not touch `reference/`.
- Do not copy proprietary Jianying assets.
- Do not add icon packages.
- Do not introduce renderer-owned FFmpeg/render graph/GPU/cache/timeline semantics.
- Do not count playhead movement as playback completion.
