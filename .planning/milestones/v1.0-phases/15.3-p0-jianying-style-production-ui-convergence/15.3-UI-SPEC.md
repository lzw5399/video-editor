# Phase 15.3 — UI Design Contract

> Visual and interaction contract for converting the desktop editor into a production Jianying-style workspace. Generated inline from user decisions and existing reference docs; verified against the six UI dimensions below.

| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline controls, compact chips |
| sm | 8px | Button padding, toolbar gaps, card interiors |
| md | 12px | Dense panel rows, inspector controls |
| lg | 16px | Panel padding, modal sections |
| xl | 24px | Entry shell groups, modal content gaps |
| 2xl | 32px | Entry shell primary grouping only |

Exceptions: 1px borders and timeline ruler/segment subpixel CSS calculations are allowed.

| Role | Size | Weight | Line Height |

Rules:

- Letter spacing is `0`.
- Compact panels must not use hero-scale type.
- Text must fit its container at 1280x800 and 1120x720 without overlap.

## Layout Contract

- Default launch shows a project entry state first. It includes product mark, `新建项目`, `打开项目`, optional recent project list, and no material import controls.
- Editor workspace uses five visible zones:
  - top product/category/export bar;
  - left resource panel;
  - center preview/player;
  - right contextual inspector;
  - bottom timeline.
- Top primary categories remain the only primary navigation. Do not duplicate primary categories inside the left panel.
- Export lives in the top-right product bar and opens a modal. Preview must not contain a permanent export panel in production mode.
- Preview controls include canvas, timecode, play/pause, previous/next frame, fit, aspect, and fullscreen. Numeric preview time is developer diagnostics only.
- Timeline default toolbar includes undo/redo, play/pause, split at playhead, delete, add-track menu, snapping, zoom, and selected direct-edit tools. Numeric move/split/trim inputs are developer diagnostics only.
- Right inspector is contextual: no selection shows draft/canvas summary and edit button; video/image selection shows visual controls; text selection shows text controls; audio selection shows audio controls.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Project entry primary CTA | `新建项目` |
| Project entry secondary CTA | `打开项目` |
| Import CTA | `导入素材` |
| Export CTA | `导出` |
| Export modal title | `导出` |
| Empty project body | `先新建或打开项目，再导入素材开始剪辑。` |
| Empty material body | `导入视频、图片或音频素材后，会显示在这里。` |
| Realtime unavailable | `实时预览不可用` plus one product-safe reason |
| Destructive confirmation | `{action}: 确认后无法直接恢复，是否继续？` |

Forbidden default copy: `FFmpeg`, `ffprobe`, `backend`, `Mock`, `runtime`, `artifact`, `cache`, `requestPreviewFrame`, `生成预览片段`, raw local paths, raw diagnostics, graph/cache internals.

## Interaction Contract

- Project entry must use product-level create/open flows before import.
- Material import uses system picker in production. Manual file path import is developer diagnostics only.
- Playback passes only with visible preview pixel motion and `renderGraphGpuComposited` evidence.
- Modal/dropdown/open-section states must be tested, including export advanced fields and draft parameter editing.
- Unsupported visible controls must be hidden, disabled with product-safe copy, or moved behind developer diagnostics; they cannot appear to work through fallback.
- Renderer controls emit typed commands or platform requests; accepted Rust command responses remain the source of draft/timeline truth.
