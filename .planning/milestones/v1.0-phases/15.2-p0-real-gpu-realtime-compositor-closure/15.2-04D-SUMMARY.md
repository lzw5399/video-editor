Additional probe results:

- Playwright-launched Electron process is visible to System Events but reports `frontmost=false` and `count of windows=0`.
- Electron `focus({ steal: true })`, `BrowserWindow.focus()`, `moveTop()`, `setAlwaysOnTop()`, `app.dock.show()`, and `setAccessibilitySupportEnabled(true)` did not make it frontmost.
- AppleScript `set frontmost`, `tell application id "com.github.Electron" to activate`, and packaged `tell application "Video Editor" to activate` did not make the process frontmost.
- LaunchServices `open -n` for both Electron.app and packaged `Video Editor.app` still reported `frontmost=false`.
- System Events and CoreGraphics coordinate clicks did not flip the process to frontmost in this automation session.

This is now a foreground/AppKit activation blocker in the automated desktop environment, not a render graph, native texture sampling, layer attachment, hidden-view, or no-fallback issue.
