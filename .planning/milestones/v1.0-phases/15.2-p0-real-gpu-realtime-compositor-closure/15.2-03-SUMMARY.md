# Plan 15.2-03 Summary

## Outcome

Imported material frames can now enter realtime preview through a Rust-owned
`MediaIoFrameProvider` path. Product compositor input accepts only proven
native texture handles compatible with the preview GPU device. CPU frame
handles, platform-opaque frames, FFmpeg CPU frames, and preview artifacts fail
closed as unavailable diagnostics instead of satisfying product preview.

Realtime frame requests also carry optional audio preview sync state. The Rust
runtime rejects presentation when audio session generation, target time,
buffering, or status is stale/unavailable, so video frames cannot be counted as
success while audio state is out of sync.
