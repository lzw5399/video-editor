# Phase 07 — UI Design Contract

> Visual and interaction contract for Phase 07. Generated by gsd-ui-researcher for the Electron desktop editor project canvas space and coordinate system.

---

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, scrollbar thickness, compact control gaps, status chip gaps |
| sm | 8px | Inspector row gaps, segmented button padding, inline field gaps |
| md | 16px | Default section padding and compact empty-state padding |
| lg | 24px | Preview surround breathing room at wider desktop widths |
| xl | 32px | Standard control height and icon-button rhythm |
| 2xl | 48px | Existing top feature bar height |
| 3xl | 64px | Maximum top category item width at 1280px and wider |

Exceptions:

- Workspace dividers remain 1px.
- Preview canvas border remains 1px.
- Active top category underline and timeline playhead remain 2px.
- Inspector compact rows may use 28px height where existing controls already do.
- Color swatches are 28px by 28px inside 32px rows.
- Canvas preview uses `aspect-ratio` from draft canvas config instead of spacing tokens.

## Typography

Use exactly these visible UI sizes and only these two weights:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Metadata / captions | 12px | 400 | 1.4 |
| Body / controls | 13px | 400 | 1.5 |
| Panel heading / selected labels | 14px | 600 | 1.3 |
| Workspace title / monitor title | 18px | 600 | 1.2 |

Typography rules:

- Letter spacing is `0`.
- Do not scale font size with viewport width.
- Use `font-weight: 600` only for headings, selected values, important status values, and primary action text.
- Chinese labels must not wrap inside compact controls. Use ellipsis for draft names and paths, not for required control labels.
- Do not introduce hero/display typography. This remains a dense desktop editor surface.

## Inspector Canvas Contract

Location:

- Surface: `apps/desktop-electron/src/renderer/workspace/Inspector.tsx`.
- State: no selected segment, active inspector tab `画面`.
- Section title: `草稿参数`.
- This section replaces the current hard-coded `画布比例 16:9` and `画布尺寸 1920 x 1080` readouts with generated-contract-backed controls.

Controls:

| Control | Visual Contract | Interaction Contract |
|---------|-----------------|----------------------|
| `画布比例` | Compact segmented control, one row when possible; presets `16:9`, `9:16`, `1:1`, `4:3`, `3:4`, `自定义` | Selecting a preset updates width/height draft semantics only through a Rust canvas command. Active preset uses accent. |
| `画布尺寸` | Two numeric inputs labeled `宽` and `高`, plus compact derived preset text if space allows | Positive integer pixels only. Invalid local input shows inline error and disables `应用草稿参数`. |
| `帧率` | Compact select with `24 fps`, `25 fps`, `30 fps`, `50 fps`, `60 fps`; custom rational entry deferred | Stores and displays rational frame rate from Rust contracts. Do not persist floating-point FPS. |
| `画布背景` | Segmented mode row: `黑色`, `纯色`, `模糊填充`, `图片背景` | Mode changes route through Rust command. Unsupported/degraded render modes remain visible with status chips. |
| Solid color | 28px swatch plus hex input, visible only when `纯色` is active or previewing | Hex must be validated before command submission. Swatch uses existing color input styling. |
| Blur fill | Disabled or degraded row with `降级` chip if render support is incomplete | Must not silently fake support. If accepted semantically but not renderable, show a warning diagnostic. |
| Image background | Disabled row with `未接入` chip and disabled `选择素材` affordance | Visible but non-mutating until material selection is implemented. |

Default values:

- `画布比例`: `16:9`
- `画布尺寸`: `1920 x 1080`
- `帧率`: `30 fps`
- `画布背景`: `黑色`

Inspector structure:

- Keep row height between 28px and 36px.
- Label column stays 72px to 88px.
- Controls use `minmax(0, 1fr)` and never force inspector width growth.
- Group canvas controls under one bordered/elevated section; do not nest cards inside cards.
- Display existing status data below controls: `序列时长`, `轨道数量`, `素材数量`, `吸附状态`, `核心状态`.
- Command error appears below the relevant canvas section, not at the top of the whole app.

Apply behavior:

- Primary CTA is `应用草稿参数`.
- Disable CTA while a command is pending.
- Disable CTA when inputs are unchanged.
- Disable CTA when width/height/frame-rate/color validation fails.
- On success, UI updates from Rust command response only.
- Canvas commands are undoable through the existing Rust command history pattern; renderer must not create a separate local undo stack.

## Preview Monitor Contract

Location:

- Surface: `apps/desktop-electron/src/renderer/workspace/PreviewMonitor.tsx`.
- Existing fixed `16:9` canvas and ratio button must become draft-canvas-aware.

Visual contract:

- Preview canvas `aspect-ratio` follows draft canvas width/height.
- Canvas remains centered, pure black `#070707` for black background, with 1px border.
- If `纯色` background is selected and supported in preview, canvas surround/readout may reflect the solid color inside the preview area only; do not tint the whole monitor shell.
- If selected background is degraded/unsupported, preview canvas shows the supported fallback and a compact warning status.
- Title/status line includes `画布 {ratio} · {width} x {height} · {fps}`.
- The ratio control button symbol shows the current ratio, e.g. `16:9`, `9:16`, or `自定义`.
- Long status text truncates with ellipsis at 1120px; it must not overlap playback controls.

Interaction contract:

- Preview monitor may display current canvas settings and request preview frames using existing preview commands.
- Preview monitor must not directly mutate `draft.canvasConfig`.
- Fit/ratio/fullscreen controls remain shell-only unless a Rust command route exists.
- Preview/export output dimension and frame-rate expectations must come from Rust-owned draft canvas/profile, not from renderer state.

## Coordinate System Contract

This wording must be captured in Phase 07 docs and referenced by future transform/keyframe UI:

- Coordinate space: normalized canvas coordinates.
- Origin: canvas center.
- `+X`: right.
- `+Y`: up.
- `1.0` equals half of the canvas width on X and half of the canvas height on Y.
- Center is `(0, 0)`.
- Canvas right edge is `x = 1`.
- Canvas left edge is `x = -1`.
- Canvas top edge is `y = 1`.
- Canvas bottom edge is `y = -1`.

Mapping guidance:

- UI preview pixel space has top-left origin and Y down; it must convert before displaying or editing persisted values.
- Persisted transform/keyframe semantics in later phases must not use renderer pixel offsets as canonical values.
- Future pixel readouts may be shown as derived helper text, but normalized coordinates remain the shared semantic language.
- Text safe area, sticker/PIP position, segment transform, and keyframe UI must share this coordinate definition unless a later phase explicitly defines a local material-space conversion for crop or mask.

Phase 07 visible UI only needs a compact help/status line:

`坐标以画布中心为原点，X 向右，Y 向上`

Do not add transform coordinate inputs in Phase 07.

## Deferred And Degraded States

Phase 07 must keep unsupported canvas background modes visible:

| Mode | Required UI State |
|------|-------------------|
| `黑色` | Supported default |
| `纯色` | Supported if Rust command/render path supports it; otherwise visible with `降级` |
| `模糊填充` | Visible with `降级` or disabled `未接入`; no silent fake blur |
| `图片背景` | Visible with disabled `选择素材` and `未接入` |

Copy examples:

- `模糊填充 · 降级`
- `图片背景 · 未接入`
- `当前预览暂不支持该画布背景，导出/预览会显示降级结果。`

Do not hide future modes just because they are not fully renderable in Phase 07.

## Command Ownership And State

Renderer may:

- Read generated TypeScript contracts.
- Keep temporary form input state while the user edits fields.
- Build generated command envelopes through the established command helper boundary.
- Display draft canvas values from Rust command responses.

Renderer must not:

- Mutate `draft.canvasConfig`, draft tracks, segment timeranges, undo/redo stacks, snapping, render graph data, FFmpeg args, export scripts, or preview cache data.
- Persist FPS as floating-point values.
- Decide output validation width/height/fps locally.
- Implement its own canonical coordinate conversion for persisted semantics.

Expected command behavior:

- Canvas edits route through Rust-owned commands.
- Commands validate aspect ratio, dimensions, frame rate, and background.
- Mutating canvas commands are undoable.
- Command response returns updated `draft`, `commandState`, `selection`, and events using established envelope patterns.

## Accessibility And Interaction

- Every control has a Simplified Chinese accessible name.
- Segmented controls use `role="group"` or an equivalent accessible pattern with clear selected state.
- Numeric inputs expose labels `画布宽度` and `画布高度` through visible labels or `aria-label`.
- Background color control exposes `画布背景颜色`.
- Deferred buttons expose disabled state and `title`/`aria-label`, e.g. `图片背景未接入`.
- Focus states use the accent border and do not resize controls.
- Keyboard tab order follows visual order: ratio, width, height, frame rate, background mode, color/image controls, apply.
- Disabled/degraded controls must still explain status in visible Chinese copy.
