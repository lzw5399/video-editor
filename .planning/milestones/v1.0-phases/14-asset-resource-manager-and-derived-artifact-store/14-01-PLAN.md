<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Project bundle path -> SQLite/blob filesystem | Untrusted or user-controlled project paths cross into `.veproj/derived` file creation and deletion. |
| SQLite metadata -> blob filesystem | Database rows decide which blob paths are live, dirty, repairable, or collectible. |
| Renderer -> artifact commands | Renderer must remain command transport and cannot decide roots, paths, keys, fingerprints, invalidation, SQLite, or FFmpeg behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering / Information Disclosure | `paths.rs`, `BlobStore` | mitigate | Reject absolute paths, parent traversal, Windows prefixes, non-UTF8 display paths, and symlink escape before any write/delete; assert returned paths are project-relative. |
| T-14-02 | Tampering | `schema.rs` SQLite migrations | mitigate | Apply `PRAGMA foreign_keys=ON`, WAL mode, busy timeout, schema version checks, and tests that orphan dependency/chunk rows fail. |
| T-14-03 | Tampering / DoS | Atomic blob writes | mitigate | Write to temp path, sync, atomic rename, verify BLAKE3 and byte length, then update rows; startup repair demotes missing/partial blobs. |
| T-14-04 | Information Disclosure | Default desktop UI/renderer | mitigate | Initial source guards reject absolute artifact roots, SQLite names, fingerprints, dirty ranges, and graph/cache internals in renderer code. |
| T-14-SC | Tampering | Cargo package installs | mitigate | Use only research-approved crates from Package Legitimacy Audit: `rusqlite`, `blake3`, and `fs2` are OK; no [ASSUMED], [SUS], or [SLOP] package is installed. |
</threat_model>
