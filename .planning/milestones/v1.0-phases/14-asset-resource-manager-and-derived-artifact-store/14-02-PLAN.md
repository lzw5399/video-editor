<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Draft semantic references -> derived resource rows | Draft materials/fonts/effects are trusted semantic inputs, but resource rows are derived and must not mutate draft state. |
| Dirty/range/fingerprint contracts -> SQLite dependency rows | Phase 13 facts become persistence keys for invalidation and GC. |
| External material refs -> local index | External paths/URIs may be user-controlled and must not become derived blob roots or leaked default UI data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Tampering | `resource_index.rs` | mitigate | Derive stable resource IDs from semantic IDs and refs in Rust; do not let renderer or generated TS invent IDs. |
| T-14-06 | Tampering | `dependencies.rs` | mitigate | Use typed dependency kinds, checked integer ranges, foreign keys, and transactions; reject overflow/partial inserts. |
| T-14-07 | Information Disclosure | External resource refs | mitigate | Store project-relative refs when possible; avoid default UI exposure of absolute paths or source fingerprints. |
| T-14-SC | Tampering | Cargo package installs | accept | No new packages beyond Plan 14-01 approved dependencies; this plan only uses existing crate dependencies. |
</threat_model>
