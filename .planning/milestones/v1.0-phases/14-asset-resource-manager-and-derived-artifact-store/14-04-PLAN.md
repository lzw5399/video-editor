<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Generation worker -> SQLite rows | Worker progress/cancel/events update durable job/chunk and artifact state. |
| Generation worker -> BlobStore filesystem | Generated proxy, thumbnail, and waveform bytes cross into project-contained derived blob storage. |
| Cancellation request -> running work | User or service cancellation crosses into long-running artifact generation. |
| Job status -> bindings/UI | Derived lifecycle status becomes user-visible production UI data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-11 | Tampering | Job/chunk status transitions | mitigate | Use transaction-scoped state guards so terminal states are not overwritten by stale workers. |
| T-14-12 | DoS | Long-running generation | mitigate | Persist cancellation request/acknowledgement and resume state; keep full scheduler/backpressure out for Phase 16. |
| T-14-13 | Tampering / Information Disclosure | `generation.rs` to `BlobStore` | mitigate | Generated output commits through `BlobStore` only after cancellation checks, fingerprint verification, non-empty blob validation, and project-relative path storage. |
| T-14-14 | Information Disclosure | Status summaries | mitigate | Return safe labels/progress/action flags only; source guards/UI plan block SQLite paths, fingerprints, graph keys, dirty ranges, FFmpeg args, and raw logs. |
| T-14-SC | Tampering | Package installs | accept | No new external package install in this plan; path dependency wiring uses existing Rust workspace crates. |
</threat_model>
