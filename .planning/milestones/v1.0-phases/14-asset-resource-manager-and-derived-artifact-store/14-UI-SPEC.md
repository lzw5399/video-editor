# Phase 14 - UI Design Contract

> Visual and interaction contract for asset/resource status, derived artifact generation, cancellation, quota, and cache maintenance. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none; existing hand-authored React + CSS renderer |
| Preset | not applicable |
| Component library | none |
| Icon library | Existing compact text/CSS symbols only; do not add a new icon dependency for this phase |
| Font | Inter, "PingFang SC", "Microsoft YaHei", ui-sans-serif, system-ui, sans-serif |

Sources: `AGENTS.md`, `.planning/REQUIREMENTS.md`, Phase 14 `CONTEXT.md`, Phase 14 `RESEARCH.md`, `docs/ui-reference/jianying-pro/README.md`, `docs/ui-reference/jianying-pro/REGRESSION.md`, and the current renderer files under `apps/desktop-electron/src/renderer/`.

shadcn gate result: `components.json` and Tailwind/PostCSS config are absent. The current approved renderer design system is manual CSS with a compact Jianying-style desktop shell. Do not initialize shadcn or introduce registry components in Phase 14.

Phase 14 must preserve the five-zone editor workspace: top feature bar, left material/resource panel, center preview monitor, right inspector, and bottom timeline. It must add only production-facing resource/artifact status needed for user workflows. It must not add a landing page, debug dashboard, raw diagnostics banner, decorative cards, or a broad UI cleanup pass.

Visual hierarchy: when resource generation is active, the compact `资源任务` strip is the primary focal point inside the left material/resource panel. Material-row resource status is secondary and stays attached to each material. `资源维护` is tertiary and appears as a quiet maintenance control. Preview and timeline status remain supporting feedback only; they must not compete with the monitor frame or become debug-heavy status dashboards.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Status chip gaps, progress row gaps, icon-only cancel button gaps |
| sm | 8px | Material row inner gaps, compact status-strip padding, button groups |
| md | 16px | Preview monitor padding, empty-state padding, maintenance section ceiling |
| lg | 24px | Reserved for compact maintenance empty states only |
| xl | 32px | Primary/secondary action height, input height, compact maintenance toolbar height |
| 2xl | 48px | Top feature bar, timeline toolbar, material thumbnail square |
| 3xl | 64px | Material row minimum height and maximum compact status section height |

Exceptions:

- Region dividers remain 1px.
- Scrollbars remain 4px.
- Compact status chips must keep a 20px fixed height, use 4px vertical padding when padding is needed, and avoid layout shifts through fixed line-height rather than sub-grid spacing.
- Progress bars are 6px high and must not change row height while progress updates.
- Icon-only cancel controls are fixed 28x28px with Chinese `aria-label` and `title`.
- Existing desktop layout dimensions remain stable: 48px top bar, 336px left panel, 288px inspector, and 252px timeline at 1120x720 and 1280x800.

## Phase Scope

Phase 14 UI exposes resource/artifact lifecycle as concise production status. It does not expose the artifact database, cache internals, invalidation algorithms, or renderer-owned resource semantics.

Renderer may:

- Display artifact status, progress, cancellability, quota summary, GC maintenance result, and user-safe error copy returned by Rust/bindings.
- Hold local UI state for search, filter, expanded maintenance details, dismissed notices, and pending button state.
- Convert a Rust-returned displayable project-relative artifact ref into text only when developer diagnostics are explicitly enabled or when a user-facing workflow needs that relative ref.
- Submit generated command envelopes for status refresh, generation retry/resume/cancel, and cache cleanup.

Renderer must not:

- Compute artifact roots, blob paths, cache keys, fingerprints, graph node IDs, dirty ranges, invalidation scopes, SQLite queries, quota totals, or GC deletion candidates.
- Show SQLite paths, absolute cache roots, FFmpeg/probe internals, graph fingerprints, cache keys, artifact fingerprints, invalidation internals, or raw diagnostics in default production UI.
- Directly mutate draft materials, tracks, segments, timeranges, keyframes, preview cache entries, waveform/proxy/thumbnail rows, or artifact job state.
- Build FFmpeg commands, render graphs, export scripts, resource manifests, sync manifests, or derived artifact indexes in TypeScript.

## Interaction Contract

### Material Panel Resource Status

Extend each material row with a compact artifact status line below existing metadata. The line must fit inside the existing material list rhythm without changing row height on hover or progress updates.

Required chips:

| Artifact | Labels |
|----------|--------|
| Thumbnail | `缩略图`, plus `资源就绪` / `生成中` / `待刷新` / `生成失败` |
| Waveform | `波形`, plus `资源就绪` / `等待生成` / `生成中` / `生成失败` |
| Proxy | `代理`, plus `资源就绪` / `等待生成` / `生成中` / `可继续` / `已取消` |

Chip rules:

- Ready uses success text only, not a filled green pill.
- Generating uses accent text plus an adjacent 6px progress bar when progress is known.
- Dirty/waiting/resumable use warning text.
- Failed uses destructive text and exposes `重新生成` as a compact secondary action if Rust reports the action is available.
- The material row never shows raw artifact paths, source fingerprints, graph fingerprints, cache keys, or DB row IDs.

### Resource Task Strip

Add a compact `资源任务` strip in the media panel, above the material list and below filters. It summarizes active derived generation without becoming a scheduler dashboard.

Layout:

- Maximum height: 64px by default.
- Show at most three active rows. Additional rows collapse into `另有 {N} 个资源任务`.
- Each row: artifact label, material display name or safe operation label, status copy, progress bar, optional `取消生成` icon/text button.
- Progress uses Rust-returned per-mille/percent facts only. If progress is unknown, show an indeterminate text state `生成中` without fake percentages.

Interaction:

- `取消生成` sends a generated cancellation command and immediately changes the row to `正在取消` while awaiting Rust response.
- Cancellation completion shows `已取消`; the row may remain until the next status refresh or user dismissal.
- Failed rows show `重新生成` only when Rust returns a retryable status.
- Resumable rows show `继续生成` only when Rust returns a resumable status.

### Preview And Timeline Status

Preview production UI remains user-safe:

- Default preview status line may show `预览资源生成中`, `预览就绪`, `预览待刷新`, `生成失败`, or `已取消`.
- The preview canvas still shows the actual preview frame when available and the existing black placeholder when unavailable.
- Developer diagnostics may keep a compact artifact panel, but Phase 14 must replace absolute artifact paths with Rust-returned displayable project-relative refs. Default production UI must not show artifact refs.
- Timeline audio segments may use waveform readiness returned from Rust for display, but waveform data remains derived and non-canonical. If waveform is missing, preserve the current stable placeholder stripe rather than changing segment height.

### Cache Maintenance And Quota

Add one compact `资源维护` section in the media panel or draft-level inspector state. It is a maintenance control, not a file browser.

Required fields:

| Field | Copy / Format |
|-------|---------------|
| Summary normal | `缓存空间正常 · {usedLabel}` |
| Summary warning | `缓存空间偏高 · {usedLabel}` |
| Cleanup estimate | `可清理 {reclaimableLabel}` |
| Last result | `缓存清理完成 · 已释放 {releasedLabel}` |
| Error | `缓存清理失败：请关闭其他窗口后重试。` |

Rules:

- `usedLabel`, `reclaimableLabel`, and `releasedLabel` are display strings returned or approved by Rust/bindings; TypeScript must not calculate quota totals from paths.
- `清理缓存` requires confirmation using the copy in the Copywriting Contract.
- Cleanup may show a progress bar only when Rust reports progress; otherwise show `正在清理缓存`.
- The maintenance section must explicitly reassure that source media is not deleted: `不会删除原始素材`.
- Do not expose blob folders, database file names, tombstones, manifests, or deletion candidates.

### Developer Diagnostics Exception

When `showDeveloperDiagnostics` is true, a compact diagnostics surface may show:

- Displayable project-relative artifact refs returned by Rust.
- Artifact kind, status, generation label, and safe error category.

Even in diagnostics mode, do not show:

- Absolute paths outside the project bundle.
- SQLite file paths or SQL/table names.
- Raw fingerprints, cache keys, graph node IDs, dirty ranges, FFmpeg command lines, probe JSON, or raw runtime logs.
