## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Text/audio command payload -> semantic draft | User-provided text, style, audio volume, and mute requests become persisted draft semantics. |
| Command transaction -> returned draft/state | Text/audio edits commit only after validation succeeds and history is updated after commit. |
| Binding route -> `draft_commands` | Native binding must delegate text/audio command behavior to the pure command crate. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Malformed text/audio command payloads | mitigate | Strict generated payloads, schema exports, and binding invalid-payload behavior through `CommandEnvelope`. |
| T-03-02 | Tampering/DoS | Text/audio source and target timeranges | mitigate | Reuse checked microsecond timerange helpers and reject zero/overflow/overrun cases. |
| T-03-03 | Tampering | Locked track, incompatible audio/text target, or mute mutation | mitigate | Reject locked-track mutation and incompatible track/material pairs before commit. |
| T-03-04 | Repudiation/Tampering | Rejected text/audio command history mutation | mitigate | Use clone/validate/commit and push history only after successful validation; tests cover rejected text/audio edits. |
| T-03-05 | Elevation of Privilege/Tampering | Renderer bypass of Rust text/audio semantics | mitigate | Binding routes generated commands to `draft_commands` and does not implement text/audio semantics locally. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install is planned. Stop for human legitimacy verification if a new dependency is proposed. |
</threat_model>
