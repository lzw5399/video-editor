## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External adapter to draft model | Provider/native effect references enter compatibility reports and must not become internal supported semantics. |
| Renderer to generated TS contracts | UI sees type shapes but cannot evaluate capability semantics. |
| Render graph to preview/export | Typed support states cross into runtime layers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-04 | Tampering | `draft_model::effects` | mitigate | Use tagged serde enums, `deny_unknown_fields`, and schema tests for typed parameters and support states. |
| T-19-05 | Tampering | Provider IDs | mitigate | Model external references as report-only compatibility data, never as supported `EffectKind`/`TransitionKind`. |
| T-19-06 | Repudiation | Capability reports | mitigate | Support states include explicit preview/export status and reason fields consumed by graph/runtime/compiler diagnostics. |
| T-19-SC | Tampering | dependency stack | accept | No package installs or upgrades in this plan; existing lockfiles are used. |
</threat_model>
