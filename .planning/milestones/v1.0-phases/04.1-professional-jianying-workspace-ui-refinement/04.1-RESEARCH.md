- Persisted semantics still use integer microseconds/rational frame rates. UI may format time for display only.
- All user-visible copy is Simplified Chinese and uses Jianying-style editing terms.
- If adding icons, prefer `lucide-react` generic icons; do not copy Jianying icons.

## Summary

Phase 04.1 is a refinement of an existing React/Electron workspace, not a new shell. The current code already has the exact file boundaries the phase needs: `WorkspaceShell.tsx` owns the five-region layout and top categories, `FeaturePanel.tsx` owns resource/function pages, `PreviewMonitor.tsx` is a tiny shell, `Inspector.tsx` owns selected/no-selected controls, `Timeline.tsx` owns toolbar/ruler/track/segment display, `viewModel.ts` owns display-only derivations, and `commandHelpers.ts` plus `App.tsx` preserve the Rust command path. [VERIFIED: codebase grep]

The main planning risk is not layout complexity; it is boundary drift while making the UI denser. `App.tsx` currently builds command envelopes with source/target timeranges for existing Rust commands, while `scripts/phase4-source-guards.sh` blocks direct renderer mutation and render/media artifact behavior. The plan must preserve this distinction: command payload construction through existing helpers is allowed, direct `draft.tracks` or `track.segments` mutation and preview/export implementation are not. [VERIFIED: App.tsx + commandHelpers.ts + scripts/phase4-source-guards.sh]

**Primary recommendation:** Implement four slices matching the roadmap: top/left, preview/inspector, timeline, tests/guards. Add `lucide-react` only if the planner accepts dependency and lockfile churn; otherwise use compact accessible text/CSS symbols for this phase, but do not hand-draw custom SVGs or copy proprietary icons. [VERIFIED: ROADMAP.md + UI-SPEC.md + lucide.dev + npm registry + slopcheck]

### Pattern 1: Geometry Tests Use Boxes, Not Pixel Snapshots

**What:** Reuse `expectStableBox`, `expectNoOverlap`, and viewport resizing; add checks for category label wrapping, preview canvas aspect ratio, toolbar/ruler separation, and internal inspector scrolling. [VERIFIED: workspace.spec.ts]  
**When to use:** Required for 1280x800 and 1120x720 layout correctness without brittle screenshot diffs. [VERIFIED: UI-SPEC.md]  
**Example:**

```ts
// Source: apps/desktop-electron/tests/workspace.spec.ts + Playwright locator.boundingBox docs
const topBox = await expectStableBox(page.locator('[aria-label="顶部功能区"]').first(), "顶部功能区 1120x720");
const timelineBox = await expectStableBox(page.locator('[aria-label="时间线"]'), "时间线 1120x720");
expectNoOverlap(topBox, timelineBox, "顶部功能区", "时间线");
```

### Pattern 2: Icon Buttons Stay Accessible Through Button Labels

**What:** If `lucide-react` is used, icons should be decorative inside buttons and the button should carry the Chinese `aria-label`/`title`. Lucide official docs say icon buttons should usually label the button rather than the icon. [CITED: https://lucide.dev/guide/react/advanced/accessibility]  
**When to use:** Top categories, timeline toolbar, preview transport, track headers, keyframe placeholders. [VERIFIED: UI-SPEC.md]  
**Example:**

```tsx
// Source: Lucide accessibility docs + UI-SPEC Chinese label contract
<button type="button" className="icon-button" aria-label="撤销" title="撤销" onClick={onUndo}>
  <Undo2 aria-hidden="true" size={16} />
</button>
```

### Pattern 3: Display-Only View Models Are Allowed

**What:** `viewModel.ts` already formats microseconds and derives timeline row display from Rust-owned draft state; this is the right place for category icon metadata, track icon labels, material filter labels, draft stats, and CSS class names. [VERIFIED: viewModel.ts]  
**When to use:** Any read-only label/count/class calculation needed by the professional UI. [VERIFIED: UI-SPEC.md]  
**Do not use:** To compute accepted timeline edit semantics, snapping candidates, undo/redo behavior, preview cache keys, or render artifacts. [VERIFIED: AGENTS.md + CONTEXT.md]

### Anti-Patterns to Avoid

- **Pixel-perfect Jianying clone:** The phase allows black-box observation of layout patterns but forbids copying proprietary assets, icons, exact artwork, presets, resources, or trade dress. [VERIFIED: CONTEXT.md]
- **Renderer semantic mutation:** Do not assign/splice/sort `draft.tracks` or `track.segments`, assign timeranges, or own undo/redo/snapping state in renderer code. [VERIFIED: AGENTS.md + scripts/phase4-source-guards.sh]
- **Preview/export leakage into UI shell:** Do not add FFmpeg, render graph, preview cache, thumbnail generation, waveform generation, proxy, or export logic to renderer code. [VERIFIED: CONTEXT.md + scripts/phase4-source-guards.sh]
- **Text-heavy toolbar regression:** Timeline and preview controls should be icon-first with stable dimensions and accessible labels. [VERIFIED: UI-SPEC.md]
- **Nested card/dashboard style:** The UI should remain a dense operational editor; existing cards should become compact rows/surfaces, not marketing/dashboard panels. [VERIFIED: UI-SPEC.md]

### Pitfall 2: Toolbar Wrapping at 1120x720

**What goes wrong:** Existing `.transport-strip` is 80px high and wraps many text buttons/inputs; the UI-SPEC wants 40-48px toolbar plus 28px ruler. [VERIFIED: styles.css + UI-SPEC.md]  
**Why it happens:** Text-heavy controls consume width and force wrapping. [VERIFIED: Timeline.tsx + styles.css]  
**How to avoid:** Use fixed 32px icon buttons, compact grouped numeric fields, and an overflow-safe toolbar layout; assert every toolbar child stays inside strip and ruler/first track are not covered. [VERIFIED: workspace.spec.ts + UI-SPEC.md]  
**Warning signs:** `expectTimelineControlsInsideStrip` fails or first track row has a top coordinate less than toolbar/ruler bottom. [VERIFIED: workspace.spec.ts]

### Pitfall 3: Guard False Positives Around Command Payloads

**What goes wrong:** Over-broad guard patterns can flag valid `sourceTimerange`/`targetTimerange` command payload construction in `App.tsx` or `commandHelpers.ts`. [VERIFIED: App.tsx + commandHelpers.ts]  
**Why it happens:** Renderer is allowed to build typed command envelopes, but not directly mutate existing draft/track/segment objects. [VERIFIED: AGENTS.md + commandHelpers.ts]  
**How to avoid:** Guard for assignment/mutation patterns and artifact strings, not all mentions of timerange fields. Keep command helper construction as the allowed path. [VERIFIED: scripts/phase4-source-guards.sh]  
**Warning signs:** Guard blocks object literals passed to `buildAddSegmentCommand`, `buildTrimSegmentCommand`, `buildAddTextSegmentCommand`, or `buildAddAudioSegmentCommand`. [VERIFIED: App.tsx]

### Pitfall 4: Preview Shell Accidentally Becomes Preview Pipeline

**What goes wrong:** Adding fit/ratio/fullscreen/status controls can invite preview cache, frame, thumbnail, waveform, or render-graph code in renderer. [VERIFIED: CONTEXT.md]  
**Why it happens:** The UI needs realistic slots before Phase 05, but Phase 05 owns actual preview/export. [VERIFIED: ROADMAP.md]  
**How to avoid:** Use disabled/shell-only controls, small status text, and source guards for `ffmpeg`, `ffprobe`, `renderGraph`, `previewCache`, `thumbnail`, `waveform`, `proxy`, and export script terms. [VERIFIED: UI-SPEC.md + scripts/phase4-source-guards.sh]  
**Warning signs:** New renderer code imports Node/Electron APIs, constructs paths, or references media generation terms. [VERIFIED: scripts/phase4-source-guards.sh]

### Pitfall 5: Accessibility Regressions From Icon-Only Controls

**What goes wrong:** Replacing Chinese text buttons with icons can remove accessible names. [VERIFIED: UI-SPEC.md]  
**Why it happens:** SVGs are often decorative and may be hidden from assistive technologies by default. Lucide docs state icons default to `aria-hidden="true"`. [CITED: https://lucide.dev/guide/react/advanced/accessibility]  
**How to avoid:** Put Chinese `aria-label` and `title` on every icon-only button; keep `aria-pressed` for top category and selected segment buttons. [VERIFIED: UI-SPEC.md]  
**Warning signs:** `getByRole("button", { name: "撤销" })` or similar locators fail after icon conversion. [VERIFIED: workspace.spec.ts pattern]
